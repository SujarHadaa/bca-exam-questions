2. What is the importance of Garbage collection in .NET framework? Explain .NET Framework architecture with suitable diagram.

Part A: Importance of Garbage Collection in .NET

Garbage Collection (GC) is an automatic memory management system in the .NET framework running under the Common Language Runtime (CLR). It automatically allocates and frees up the memory space used by objects that are no longer referenced by the program.

Key Benefits & Importance:

  • No Manual Memory Deallocation: Programmers do not need to explicitly release memory (no delete or free keywords needed like in C/C++), significantly speeding up development time.
  • Prevents Memory Leaks: By safely clearing out unreachable and unreferenced objects, GC prevents memory from getting completely exhausted over time.
  • Heap Compaction & Optimization: After removing dead objects, GC shifts the remaining live objects together (compaction). This makes the free memory contiguous, allowing future memory allocations to be extremely fast.
  • Memory Safety & Integrity: Ensures an object cannot accidentally use or corrupt the memory assigned to another object.

⚡ GC Generations (Bonus Info for Exams)

To optimize performance, GC organizes memory into three generations:

  • Generation 0: Holds short-lived, newly created objects (like temporary variables). GC checks this generation most frequently.
  • Generation 1: Acts as a buffer for objects that survive Gen 0. They are mid-lived objects.
  • Generation 2: Holds long-lived objects (like static variables or global application data). GC checks this rarely.

Part B: .NET Framework Architecture

The .NET Framework is a multi-layered execution environment that sits between the operating system and the application code. It compiles source code into an intermediate format, and later into machine code, allowing the program to run efficiently and securely on any compatible platform.

Figure: Layered Architecture of .NET Framework

Major Core Components:

  • CLR (Common Language Runtime): The "heart" or execution engine of the .NET Framework. Its responsibilities include:
    • Memory Management: Handled by Garbage Collection.
    • Code Execution: Contains the JIT (Just-In-Time) compiler, which translates MSIL (Microsoft Intermediate Language) into native machine code (0s and 1s) specific to the underlying OS.
    • Exception Handling: Catches runtime errors to prevent application crashes.
  • FCL (Framework Class Library) & BCL (Base Class Library): A massive collection of ready-to-use, reusable classes, interfaces, and methods. For instance, the BCL handles basic operations (File I/O, Strings, Collections), while the broader FCL handles advanced things like Database connections (ADO.NET) or Web apps (ASP.NET).
  • CLS (Common Language Specification): A set of minimum standardized rules that all .NET languages must follow. It guarantees Cross-Language Interoperability (e.g., a class written in C# can be inherited perfectly in VB.NET).
  • CTS (Common Type System): Defines how data types are declared and managed uniformly across the framework. For example, a 4-byte integer is int in C# and Integer in VB.NET, but both map to the common System.Int32 type underneath.
3. What is variable size array? Write a C# program to create multidimensional array to store the marks of three student in different subjects. First student has marks of 3 subjects, Second student has marks of 4 subjects and Third student has marks of 2 subjects. Display the subject marks and average marks for each student.

Part A: Variable Size Array (Jagged Array)

In C#, a variable size array is called a Jagged Array. It is essentially an "array of arrays." Unlike a regular multidimensional array (like a grid where all rows have the same number of columns), each row in a jagged array can have a different length (variable size).

  • Memory Efficient: It only allocates as much memory as needed for each row.
  • Flexibility: Very useful for real-world scenarios, like storing marks of students who have taken a different number of subjects.

Part B: C# Program for Student Marks

Below is the C# program that uses a jagged array to store and display the marks and their average for three students with a different number of subjects: 

using System;

class Program
{
    static void Main()
    {
        // 1. Create a jagged array (variable size array) for 3 students
        int[][] studentMarks = new int[3][];

        // 2. Initialize the marks for each student
        // First student has 3 subjects
        studentMarks[0] = new int[3] { 85, 90, 78 };
        
        // Second student has 4 subjects
        studentMarks[1] = new int[4] { 70, 88, 92, 80 };
        
        // Third student has 2 subjects
        studentMarks[2] = new int[2] { 85, 95 };

        // 3. Display marks and calculate average for each student
        for (int i = 0; i < studentMarks.Length; i++)
        {
            Console.WriteLine($"Student {i + 1} Marks:");
            
            int sum = 0;
            for (int j = 0; j < studentMarks[i].Length; j++)
            {
                Console.Write(studentMarks[i][j] + " ");
                sum += studentMarks[i][j];
            }
            
            double average = (double)sum / studentMarks[i].Length;
            Console.WriteLine($"\nAverage Marks: {average:F2}\n");
        }
        
        Console.ReadLine();
    }
}
4. Explain concept of static class and static constructor with suitable program.

Part A: Concept of Static Class

A Static Class in C# is a class that cannot be instantiated (you cannot use the new keyword to create an object of it). It is primarily used to group related utility or helper methods that do not require an object state to function.

  • No Objects: You cannot create instances of a static class.
  • Static Members Only: It can only contain static methods, static fields, and static properties.
  • Sealed by Default: A static class cannot be inherited by other classes.
  • Direct Access: Members are accessed directly using the class name (e.g., Math.Sqrt()).

Part B: Concept of Static Constructor

A Static Constructor is a special constructor used to initialize static data or perform a particular action that needs to be performed only once. It is called automatically before the first instance is created or any static members are referenced.

  • Automatic Invocation: The CLR calls it automatically. You cannot call it directly.
  • No Parameters or Modifiers: It cannot take parameters and cannot have access modifiers like public or private.
  • Runs Once: It executes exactly once per class, no matter how many objects of the class are created.

Part C: Suitable C# Program

Below is a C# program demonstrating both a static class and a static constructor: 

using System;

// 1. Static Class
static class MathUtility
{
    // Static field
    public static double Pi;

    // 2. Static Constructor (No access modifier, no parameters)
    static MathUtility()
    {
        Pi = 3.14159;
        Console.WriteLine("Static Constructor Called! (This runs only once)");
    }

    // Static Method
    public static double CalculateCircleArea(double radius)
    {
        return Pi * radius * radius;
    }
}

class Program
{
    static void Main()
    {
        Console.WriteLine("Main method started.");
        
        // Note: MathUtility obj = new MathUtility(); // This will cause an ERROR!

        // Accessing static method directly via Class Name
        // The static constructor is called right before this line executes.
        double area1 = MathUtility.CalculateCircleArea(5.0);
        Console.WriteLine($"Area of Circle 1: {area1}");
        
        // Calling it again to prove static constructor does NOT run a second time.
        double area2 = MathUtility.CalculateCircleArea(10.0);
        Console.WriteLine($"Area of Circle 2: {area2}");
        
        Console.ReadLine();
    }
}
5. What is named argument? Explain with suitable program, how to create alias for namespace in C#?

Part A: Concept of Named Argument

In C#, Named Arguments allow you to pass arguments to a method by explicitly associating them with their corresponding parameter names, rather than relying on their position in the parameter list.

  • Order Independence: You do not need to remember the order of parameters. You can pass them in any order.
  • Improved Readability: It makes the code much easier to read, especially when dealing with methods that take multiple parameters of the same type.

Part B: Concept of Namespace Alias

A Namespace Alias allows you to create a short, alternative name (alias) for a long or conflicting namespace (or class). This is extremely useful when two different namespaces have classes with the exact same name, as it resolves ambiguity.

  • Resolves Conflicts: Prevents naming collisions between identical class names in different namespaces.
  • Shorter Code: Provides a shorthand identifier for long, deeply-nested namespaces.

Part C: Suitable C# Program

Below is a C# program demonstrating both Named Arguments and Namespace Aliasing: 

using System;

// 1. Creating a Namespace Alias
// 'Printer' is now an alias for the 'System.Console' class
using Printer = System.Console;

class Program
{
    // A method with multiple parameters
    static void DisplayStudentInfo(string name, int age, string city)
    {
        Printer.WriteLine($"Name: {name}, Age: {age}, City: {city}");
    }

    static void Main()
    {
        Printer.WriteLine("--- Named Arguments Demo ---");

        // Traditional way (Positional Arguments):
        // DisplayStudentInfo("Alice", 20, "Kathmandu");

        // 2. Using Named Arguments
        // The order of parameters does not matter!
        DisplayStudentInfo(city: "Pokhara", name: "Bob", age: 22);
        DisplayStudentInfo(age: 21, city: "Lalitpur", name: "Charlie");
        
        Printer.WriteLine("\nNotice how 'Printer' was used instead of 'Console' due to the alias!");
        
        Printer.ReadLine();
    }
}
6. What is static binding? Write a C# program to add and subtract two complex number using binary operator overloading.

Part A: Static Binding

Static Binding (also known as early binding or compile-time polymorphism) occurs when the compiler resolves the method or function calls at compile time. In other words, the decision of which method implementation to execute is made based on the reference type of the object during compilation, rather than its actual type at runtime.

  • Performance: It is faster than dynamic binding because the method resolution happens beforehand.
  • Examples: Method Overloading and Operator Overloading are prime examples of static binding.

Part B: Binary Operator Overloading (Complex Numbers)

Below is a C# program demonstrating how to use static binding through binary operator overloading to add and subtract two complex numbers (which have real and imaginary parts): 

using System;

class Complex
{
    private double real;
    private double imag;

    // Constructor to initialize the complex number
    public Complex(double r, double i)
    {
        real = r;
        imag = i;
    }

    // 1. Overloading the binary '+' operator
    public static Complex operator +(Complex c1, Complex c2)
    {
        return new Complex(c1.real + c2.real, c1.imag + c2.imag);
    }

    // 2. Overloading the binary '-' operator
    public static Complex operator -(Complex c1, Complex c2)
    {
        return new Complex(c1.real - c2.real, c1.imag - c2.imag);
    }

    // Helper method to display the complex number cleanly
    public void Display()
    {
        if (imag < 0)
            Console.WriteLine($"{real} - {Math.Abs(imag)}i");
        else
            Console.WriteLine($"{real} + {imag}i");
    }
}

class Program
{
    static void Main()
    {
        Complex c1 = new Complex(5.0, 3.0);
        Complex c2 = new Complex(2.0, 1.0);

        Console.WriteLine("--- Complex Numbers ---");
        Console.Write("First Complex Number  (c1): ");
        c1.Display();
        
        Console.Write("Second Complex Number (c2): ");
        c2.Display();
        Console.WriteLine();

        // Using the overloaded operators
        Complex sum = c1 + c2;       // Internally calls the overloaded '+' method
        Complex diff = c1 - c2;      // Internally calls the overloaded '-' method

        Console.Write("Sum (c1 + c2)        : ");
        sum.Display();

        Console.Write("Difference (c1 - c2) : ");
        diff.Display();

        Console.ReadLine();
    }
}
7. Why Delegate is used in C#? Write a C# program to select odd and divisible by 3 number from list of number (1-30) using LINQ query.

Part A: Why Delegate is used in C#?

A Delegate in C# is a type that safely encapsulates a method, similar to a function pointer in C or C++, but unlike function pointers, delegates are object-oriented, type-safe, and secure. They are used for the following key reasons:

  • Callback Methods: Delegates allow methods to be passed as parameters to other methods, making it possible to define callback functions.
  • Event Handling: They are the foundation of events in C#. The publisher of an event uses a delegate to call the subscribers' event handlers.
  • LINQ Queries: Under the hood, LINQ heavily relies on delegates (like Func, Action, and Predicate) to pass conditions to querying methods (e.g., Where clause).
  • Loose Coupling: The caller does not need to know the actual method or object it is invoking, promoting a decoupled architecture.

Part B: LINQ Query (Odd & Divisible by 3)

Below is a C# program that generates a list of numbers from 1 to 30, and then uses a LINQ query to select only those numbers that are both odd and divisible by 3: 

using System;
using System.Collections.Generic;
using System.Linq;

class Program
{
    static void Main()
    {
        // 1. Create a list of numbers from 1 to 30
        List<int> numbers = new List<int>();
        for (int i = 1; i <= 30; i++)
        {
            numbers.Add(i);
        }

        // 2. Write LINQ Query to select numbers:
        // Condition 1: Odd number (num % 2 != 0)
        // Condition 2: Divisible by 3 (num % 3 == 0)
        var selectedNumbers = from num in numbers
                              where num % 2 != 0 && num % 3 == 0
                              select num;

        // Alternatively using Method Syntax:
        // var selectedNumbers = numbers.Where(n => n % 2 != 0 && n % 3 == 0);

        // 3. Display the result
        Console.WriteLine("Numbers from 1 to 30 that are ODD and DIVISIBLE BY 3:");
        foreach (int n in selectedNumbers)
        {
            Console.Write(n + " ");
        }
        
        Console.WriteLine();
        Console.ReadLine();
    }
}

a) Copy Constructor

A Copy Constructor is a parameterized constructor that takes an object of the same class type as a parameter. Its primary purpose is to initialize a new object by copying the variables and data from an existing object.

  • Manual Implementation: Unlike languages like C++, C# does not provide a default copy constructor. If you need one, you must manually define it.
  • Usage: It is highly useful when you want to duplicate an object but want to modify the new object without affecting the original one.
class Student
{
    public string Name;
    
    // Copy Constructor
    public Student(Student existingStudent)
    {
        this.Name = existingStudent.Name; // Copies the value
    }
}

b) Indexer

An Indexer allows instances of a class or struct to be indexed exactly like arrays. It enables client code to access elements within an object using the square bracket notation [].

  • The this Keyword: Indexers are always defined using the this keyword.
  • Smart Arrays: They are often referred to as "smart arrays" because they can contain internal logic in their get and set accessors, and they can be indexed by data types other than just integers (e.g., indexed by strings).
class StringCollection
{
    private string[] data = new string[10];
    
    // Indexer Declaration
    public string this[int index]
    {
        get { return data[index]; }
        set { data[index] = value; }
    }
}

c) Lambda Expression

A Lambda Expression is a short, concise, inline anonymous method. It is used to create delegates or expression tree types locally, heavily reducing the amount of code needed.

  • The Lambda Operator (=>): The expression uses the => operator, which is read as "goes to". The left side specifies input parameters, and the right side holds the expression or statement block.
  • Core to LINQ: Lambda expressions are used extensively in LINQ queries to quickly define filtering, sorting, or projection logic (e.g., Where(x => x > 10)).
// Syntax: (input-parameters) => expression

// Example: A lambda expression that squares a number
Func<int, int> square = x => x * x;

Console.WriteLine(square(5)); // Output: 25

Part A: Difference Between Struct and Enum

Feature Struct Enum
Definition A composite data type used to group related variables of different data types. A distinct value type consisting of a set of named integer constants.
Usage Purpose Used to create lightweight objects representing a single entity or record (e.g., a Point with X and Y coordinates). Used to represent a state or a choice from a fixed set of options (e.g., Days of the week, Status codes).
Contents Can contain fields, properties, constructors, and methods. Can only contain named constants mapped to integral numbers.
Inheritance Cannot inherit from other classes or structs (but can implement interfaces). Underlying type inherits from System.Enum.

Part B: Why do we need to handle exceptions?

An Exception is an unexpected event or error that occurs during the execution of a program (runtime error) that disrupts the normal flow of instructions. We must handle exceptions for the following reasons:

  • Prevent Application Crashes: Unhandled exceptions terminate the program abruptly. Handling them ensures the program continues to run gracefully.
  • Provide Meaningful Messages: It allows developers to show user-friendly error messages instead of technical stack traces or system error codes.
  • Safe Resource Cleanup: Using the finally block guarantees that important resources (like database connections, file handles, network streams) are released, even if an error occurs.
  • Logging & Debugging: Caught exceptions can be logged to a file or database, which is crucial for developers to trace and fix bugs.

Part C: Customized Exception Example

Below is a C# program demonstrating how to create and use a custom exception class called InvalidAgeException for a voting eligibility check. 

using System;

// 1. Creating a Custom Exception Class
// It must inherit from the base 'Exception' class
class InvalidAgeException : Exception
{
    public InvalidAgeException(string message) : base(message)
    {
    }
}

class Program
{
    // Method that checks logic and throws the customized exception
    static void CheckVotingEligibility(int age)
    {
        if (age < 18)
        {
            // Throwing our customized exception
            throw new InvalidAgeException("Age must be 18 or older to vote. Voting denied.");
        }
        else
        {
            Console.WriteLine("You are eligible to vote!");
        }
    }

    static void Main()
    {
        try
        {
            Console.Write("Enter your age: ");
            
            // Simulating user input for the example
            int userAge = 16; 
            Console.WriteLine(userAge);

            // Calling the method that might throw an exception
            CheckVotingEligibility(userAge);
        }
        catch (InvalidAgeException ex) // Handling the specific customized exception
        {
            Console.WriteLine($"[Custom Error Caught]: {ex.Message}");
        }
        catch (Exception ex) // Handling any other general exceptions
        {
            Console.WriteLine($"[General Error]: {ex.Message}");
        }
        finally
        {
            // This block always runs, ensuring resources are cleaned up
            Console.WriteLine("--- Voting eligibility check process ended ---");
        }
        
        Console.ReadLine();
    }
}
10. Define generics. Explain it with proper example. How virtual method is used to achieve dynamic binding in C#? Explain with the help of suitable program.

Part A: Generics in C#

Generics allow you to define classes, interfaces, and methods with a placeholder for the data type. This means you do not have to specify the exact data type when writing the code. Instead, the type is provided by the client code when it instantiates the class or calls the method.

  • Type Safety: Generics ensure compile-time type checking, preventing runtime casting errors.
  • Performance: They eliminate the need for boxing (converting value types to objects) and unboxing, significantly improving performance.
  • Reusability: You write the logic once and use it for any data type (e.g., List<int>, List<string>).

Example of Generics:

using System;

// A generic class using placeholder <T>
class GenericBox<T>
{
    private T data;

    public void SetValue(T value)
    {
        data = value;
    }

    public T GetValue()
    {
        return data;
    }
}

class TestGenerics
{
    static void Main()
    {
        // Using the generic class with 'int'
        GenericBox<int> intBox = new GenericBox<int>();
        intBox.SetValue(100);
        Console.WriteLine("Integer Box: " + intBox.GetValue());

        // Using the same generic class with 'string'
        GenericBox<string> stringBox = new GenericBox<string>();
        stringBox.SetValue("Hello Generics");
        Console.WriteLine("String Box: " + stringBox.GetValue());
    }
}

Part B: Dynamic Binding & Virtual Methods

Dynamic Binding (also known as late binding or runtime polymorphism) is a mechanism where the compiler does not know the exact method to call at compile time. The method to execute is determined at runtime based on the actual type of the object being referenced.

In C#, this is achieved using the virtual and override keywords:

  • virtual keyword: Added to a method in the base class to indicate that it can be overridden in a derived class.
  • override keyword: Used in the derived class to provide a specific implementation for the virtual method.
  • When a base class reference points to a derived class object, calling the virtual method triggers the derived class's overridden version.

Program Demonstrating Dynamic Binding:

using System;

// Base Class
class Animal
{
    // 'virtual' allows this method to be overridden by derived classes
    public virtual void MakeSound()
    {
        Console.WriteLine("Some generic animal sound...");
    }
}

// Derived Class 1
class Dog : Animal
{
    // 'override' replaces the base class implementation
    public override void MakeSound()
    {
        Console.WriteLine("Dog says: Woof! Woof!");
    }
}

// Derived Class 2
class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Cat says: Meow! Meow!");
    }
}

class Program
{
    static void Main()
    {
        // Base class reference variables
        Animal myAnimal1;
        Animal myAnimal2;

        // Dynamic Binding: 
        // References are of type 'Animal', but they point to 'Dog' and 'Cat' objects
        myAnimal1 = new Dog();
        myAnimal2 = new Cat();

        // The compiler sees MakeSound() on 'Animal', but at runtime,
        // the CLR checks the ACTUAL object type and calls the overridden methods.
        
        myAnimal1.MakeSound(); // Output: Dog says: Woof! Woof!
        myAnimal2.MakeSound(); // Output: Cat says: Meow! Meow!
        
        Console.ReadLine();
    }
}

Part A: Data Adapter vs Data Reader in ADO.NET

Feature Data Reader (SqlDataReader) Data Adapter (SqlDataAdapter)
Architecture Connected Architecture. It requires the connection to remain open while reading. Disconnected Architecture. It opens the connection, fetches data, and closes it.

Use Case Best for simply fetching and displaying large amounts of data quickly. Best for complex data manipulation, offline data access, and returning data to UI grids.

2. The C# Code-Behind (EmployeeForm.aspx.cs)

This code uses ADO.NET parameterized queries to securely insert the data into the database, avoiding SQL Injection. 

using System;
using System.Data.SqlClient;

namespace EMS
{
    public partial class EmployeeForm : System.Web.UI.Page
    {
        protected void Page_Load(object sender, EventArgs e)
        {
        }

        protected void btnSave_Click(object sender, EventArgs e)
        {
            // 1. Define the connection string
            string connString = "Server=myServer;Database=myDB;User Id=myUser;Password=myPass;";

            // 2. Write the INSERT query (using parameters to prevent SQL injection)
            string query = @"INSERT INTO EMS.Employee 
                             (Name, Email, PhoneNumber, Company, Department, JoiningDate, Salary) 
                             VALUES 
                             (@Name, @Email, @Phone, @Company, @Dept, @Date, @Salary)";

            // 3. Use 'using' block to automatically close connection
            using (SqlConnection conn = new SqlConnection(connString))
            {
                using (SqlCommand cmd = new SqlCommand(query, conn))
                {
                    // 4. Add values to the parameters from the TextBoxes
                    cmd.Parameters.AddWithValue("@Name", txtName.Text);
                    cmd.Parameters.AddWithValue("@Email", txtEmail.Text);
                    cmd.Parameters.AddWithValue("@Phone", txtPhone.Text);
                    cmd.Parameters.AddWithValue("@Company", txtCompany.Text);
                    cmd.Parameters.AddWithValue("@Dept", txtDept.Text);
                    cmd.Parameters.AddWithValue("@Date", txtDate.Text);
                    cmd.Parameters.AddWithValue("@Salary", Convert.ToDecimal(txtSalary.Text));

                    try
                    {
                        // 5. Open connection and execute
                        conn.Open();
                        int rowsAffected = cmd.ExecuteNonQuery();

                        if (rowsAffected > 0)
                            lblMessage.Text = "Employee saved successfully!";
                    }
                    catch (Exception ex)
                    {
                        // Display error if something goes wrong
                        lblMessage.Text = "Error: " + ex.Message;
                        lblMessage.ForeColor = System.Drawing.Color.Red;
                    }
                }
            }
        }
    }
