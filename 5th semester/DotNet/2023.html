2. What is managed code? Give reason to justify the statement "DOT NET framework is platform independent and Language interoperability".

1. Managed Code

Managed code is the code whose execution is directly managed by the Common Language Runtime (CLR) rather than the operating system.

  • When you write a program in C# or VB.NET, the compiler translates it into an intermediate language called MSIL (Microsoft Intermediate Language).
  • During execution, the CLR takes this MSIL code and converts it into machine-specific code using the JIT (Just-In-Time) compiler.
  • The CLR also provides important services to this code, such as automatic memory management (Garbage Collection), security checks, and exception handling.

2. Platform Independence in .NET

The statement "DOT NET framework is platform independent" is justified by how code is compiled and executed:

  • Two-Step Compilation: Source code is NOT directly compiled into machine code (which is hardware/OS specific). Instead, it is compiled into a universal format called MSIL.
  • Any OS with CLR: This MSIL can be taken to any operating system (Windows, Linux, macOS) that has a compatible CLR installed.
  • JIT Compilation: The local CLR on that specific machine will use its JIT compiler to translate the MSIL into native machine code on the fly. This "Write once, run anywhere" capability makes it platform independent.

3. Language Interoperability

Language interoperability means code written in one language (like C#) can interact seamlessly with code written in another language (like VB.NET or F#). This is justified by two core components of .NET:

  • Common Type System (CTS): It defines a standard set of data types across all .NET languages. For example, an integer in C# (int) and an integer in VB.NET (Integer) both map to the exact same underlying type in CTS (System.Int32). This allows them to share data without conversion issues.
  • Common Language Specification (CLS): It is a set of rules that all .NET languages must follow (like how to name methods or pass parameters) so they can understand each other.
  • Single Intermediate Language: No matter which .NET language you use, it all compiles down to the same MSIL code. Therefore, an object created in C# can easily inherit from a class written in VB.NET.

⚡ Cheat Sheet

  • Managed Code: Code that is executed and managed by the CLR (handles memory and security).
  • Platform Independent: Source Code → MSIL. Any OS with a CLR can run that MSIL using its JIT compiler.
  • Language Interoperability: Different languages share standard data types (CTS) and rules (CLS), and all compile to the same MSIL.
3. What is optional parameter? Write a C# program which stores values in two enumerations, Department and College. It uses two functions to display the data contained in Department and College enumerations.

1. Optional Parameters in C#

An optional parameter is a parameter in a method signature that has a default value assigned to it. If the caller does not provide a value for that parameter when invoking the method, the compiler automatically uses the default value. This makes calling methods more flexible and reduces the need for multiple method overloads.

  • Optional parameters must appear after all required parameters in the parameter list.
  • A default value must be a constant expression (like a number, string, or enum).

2. Program using Enumerations and Optional Parameters

Here is a C# program that defines two enumerations (Department and College) and uses optional parameters to display their values: 

using System;

namespace EnumExample
{
    // Define Enumeration for Department
    enum Department
    {
        BCA,
        BScCSIT,
        BIM,
        BBA
    }

    // Define Enumeration for College
    enum College
    {
        PatanMultipleCampus,
        AmritScienceCampus,
        BhaktapurMultipleCampus
    }

    class Program
    {
        // Function to display Department data using optional parameter
        static void DisplayDepartment(Department dept = Department.BCA)
        {
            Console.WriteLine("Selected Department: " + dept);
        }

        // Function to display College data using optional parameter
        static void DisplayCollege(College coll = College.PatanMultipleCampus)
        {
            Console.WriteLine("Selected College: " + coll);
        }

        static void Main(string[] args)
        {
            // Calling functions without arguments (uses default/optional parameter)
            DisplayDepartment();
            DisplayCollege();

            Console.WriteLine("\n--- Providing Explicit Values ---");
            
            // Calling functions with arguments
            DisplayDepartment(Department.BScCSIT);
            DisplayCollege(College.AmritScienceCampus);
            
            Console.ReadLine();
        }
    }
}

⚡ Cheat Sheet

  • Optional Parameter: void Method(int x = 10). If called as Method(), x is 10. Must be placed at the end of the parameter list.
  • Enum: A special "class" that represents a group of constants (unchangeable variables). E.g., enum Level { Low, Medium, High }.
4. What is difference between value type and reference type in C#? Explain the concept of boxing and unboxing with suitable program.

1. Value Type vs Reference Type

Feature Value Type Reference Type
Storage Location Stored on the Stack memory. Stored on the Heap memory (the stack only holds a reference/pointer to the heap).
Data Content Directly holds the actual data/value. Holds the memory address where the data is stored.
Null Values Cannot hold a null value (unless explicitly made nullable). Can easily hold a null value if it doesn't point to any object.
Examples int, float, double, struct, enum string, class, array, delegate

2. Boxing and Unboxing

Boxing is the process of converting a Value Type (like int) into a Reference Type (like object). This takes the value from the Stack and puts it inside a new "box" on the Heap. It happens implicitly (automatically).

Unboxing is the reverse process: extracting the Value Type back out of the Reference Type "box". It takes the value from the Heap and puts it back on the Stack. It requires explicit casting (manual conversion).

3. Program Illustrating Boxing and Unboxing

using System;

class Program
{
    static void Main(string[] args)
    {
        // 1. Initialize a value type variable
        int num = 100;
        
        // 2. Boxing: implicitly converting Value Type (int) to Reference Type (object)
        object obj = num; 

        // 3. Change original value (does not affect the boxed value)
        num = 200;
        
        // 4. Unboxing: explicitly converting Reference Type (object) back to Value Type (int)
        int unboxedNum = (int)obj;

        Console.WriteLine("--- Boxing and Unboxing Example ---");
        Console.WriteLine("Value Type 'num' is now: " + num);
        Console.WriteLine("Boxed Reference Type 'obj' contains: " + obj);
        Console.WriteLine("Unboxed Value Type 'unboxedNum' contains: " + unboxedNum);
        
        Console.ReadLine();
    }
}

⚡ Cheat Sheet

  • Value Type: Stack memory, holds direct data (e.g., int x = 5;).
  • Reference Type: Heap memory, holds address pointer (e.g., class, string).
  • Boxing: Value Type → Reference Type (Stack to Heap). E.g., object o = 10;
  • Unboxing: Reference Type → Value Type (Heap to Stack). E.g., int i = (int)o;
5. What is interface? Explain with suitable program, how interface can be used to perform multiple inheritance?

1. What is an Interface?

An interface in C# is a blueprint of a class. It is like a class, but it can only contain declarations of members (like methods, properties, and events) without any implementation details (no method body).

  • It acts as a strict contract: any class that implements an interface must provide the implementation for all its members.
  • By default, all members of an interface are public and abstract.
  • You cannot create an object (instantiate) directly from an interface.

2. Multiple Inheritance using Interfaces

In C#, a class can inherit from only one single base class (single class inheritance). This prevents the "Diamond Problem" (ambiguity when two parent classes have the same method). However, a class can implement multiple interfaces.

If a class needs behaviors from multiple sources, it can implement multiple interfaces, effectively achieving multiple inheritance in a safe way.

3. Program Illustrating Multiple Inheritance

using System;

// 1. Define first interface
interface IPrinter
{
    void Print();
}

// 2. Define second interface
interface IScanner
{
    void Scan();
}

// 3. Class implementing BOTH interfaces (Multiple Inheritance)
class MultiFunctionPrinter : IPrinter, IScanner
{
    // Providing implementation for Print
    public void Print()
    {
        Console.WriteLine("Printing a document...");
    }

    // Providing implementation for Scan
    public void Scan()
    {
        Console.WriteLine("Scanning a document...");
    }
}

class Program
{
    static void Main(string[] args)
    {
        // Create an object of the class
        MultiFunctionPrinter mfp = new MultiFunctionPrinter();
        
        Console.WriteLine("--- Multiple Inheritance Example ---");
        // Call methods from both interfaces
        mfp.Print();
        mfp.Scan();

        Console.ReadLine();
    }
}

⚡ Cheat Sheet

  • Interface: A contract with empty methods. Class MUST implement them. E.g., interface ICar { void Drive(); }
  • Multiple Inheritance: C# doesn't allow class A : B, C. But it allows class A : IB, IC (implementing multiple interfaces).
6. What are use of finalizer in C#? Explain with suitable program, how base keyword is used to call member function and constructor of parent class?

1. Finalizer in C#

A Finalizer (also known as a Destructor) is a special method in a class used to perform cleanup operations on unmanaged resources before an object is destroyed by the Garbage Collector (GC).

  • It is defined using the tilde (~) symbol followed by the class name.
  • It cannot have access modifiers (like public/private) and cannot take parameters.
  • You cannot call it explicitly; it is automatically invoked by the Garbage Collector when the object is no longer in use.

2. Use of 'base' Keyword

The base keyword is used within a derived (child) class to access members of the parent class. It is mainly used for two purposes:

  • Calling a Parent Constructor: It passes parameters to the base class's constructor when the derived class is instantiated.
  • Calling a Parent Member Function: It calls a method from the base class that has been hidden or overridden by a method in the derived class.

3. Program Illustrating 'base' Keyword

using System;

// Parent Class
class Animal
{
    public string name;

    // Parent Constructor
    public Animal(string n)
    {
        name = n;
        Console.WriteLine("Parent Constructor: Animal created.");
    }

    // Parent Member Function
    public virtual void Speak()
    {
        Console.WriteLine("Animal makes a sound.");
    }
}

// Child Class inheriting from Animal
class Dog : Animal
{
    // 1. Using 'base' to call the parent constructor
    public Dog(string n) : base(n)
    {
        Console.WriteLine("Child Constructor: Dog created.");
    }

    // Overriding the parent method
    public override void Speak()
    {
        // 2. Using 'base' to call the parent member function
        base.Speak(); 
        Console.WriteLine("Dog barks: Woof Woof!");
    }
}

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("--- 'base' Keyword Example ---");
        
        // This will trigger both the parent and child constructors
        Dog myDog = new Dog("Buddy");
        
        Console.WriteLine("\n--- Calling Speak Method ---");
        // This will trigger base.Speak() and then child's Speak()
        myDog.Speak();

        Console.ReadLine();
    }
}

⚡ Cheat Sheet

  • Finalizer/Destructor: ~ClassName() { }. Used to clean unmanaged resources. GC calls it automatically.
  • base Keyword: Used in child classes. base(args) calls parent constructor. base.MethodName() calls parent method.
7. What are keyword used in exception handling? Write a C# program to handle IndexOutOfRangeException and InvalidCastException.

1. Keywords in Exception Handling

Exception handling in C# is managed using four core keywords:

  • try: Used to enclose a block of code that might throw an exception.
  • catch: Used to handle the exception. You can have multiple catch blocks to handle different types of exceptions separately.
  • finally: Used to execute a block of code regardless of whether an exception is thrown or not. It is typically used for cleaning up resources.
  • throw: Used to manually trigger an exception when a specific condition is met.

2. Program Handling Multiple Exceptions

using System;

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("--- Exception Handling Example ---");
        
        try
        {
            // 1. Code that causes IndexOutOfRangeException
            int[] numbers = { 1, 2, 3 };
            Console.WriteLine("Trying to access index 5...");
            // This will fail
            int value = numbers[5]; 

            // 2. Code that causes InvalidCastException (if we reached here)
            object obj = "Hello";
            Console.WriteLine("Trying to cast string to int...");
            // This will fail
            int invalidCast = (int)obj; 
        }
        catch (IndexOutOfRangeException ex)
        {
            Console.WriteLine("Error: Array index is out of bounds!");
            Console.WriteLine("Details: " + ex.Message);
        }
        catch (InvalidCastException ex)
        {
            Console.WriteLine("Error: Invalid type casting attempted!");
            Console.WriteLine("Details: " + ex.Message);
        }
        catch (Exception ex)
        {
            // A generic catch block for any other unknown errors
            Console.WriteLine("An unexpected error occurred: " + ex.Message);
        }
        finally
        {
            Console.WriteLine("Finally block: Execution completed. Cleaning up resources if any.");
        }

        Console.ReadLine();
    }
}

⚡ Cheat Sheet

  • try: Tests the code for errors.
  • catch: Catches and handles the error.
  • finally: Always runs, error or not.
  • throw: Creates a custom error.
8. What is query syntax in LINQ? Explain the standard query operators "select", "contains", "orderBy" and "where" in LINQ with suitable examples.

1. What is Query Syntax in LINQ?

LINQ (Language Integrated Query) allows you to write queries directly in C# to retrieve data from different sources (like Arrays, Collections, Databases, XML).

Query Syntax is one of the ways to write LINQ queries. It looks very similar to SQL (Structured Query Language). A query in this syntax always begins with the from clause and ends with either a select or group clause.

2. Standard Query Operators

Here are the explanations and examples of the requested LINQ operators:

  • where: Used to filter the data based on a specific condition. It only returns items that satisfy the given condition. 
    // Example of 'where'
int[] numbers = { 1, 2, 3, 4, 5 };
var evenNumbers = from num in numbers
                  where num % 2 == 0
                  select num;
  • orderBy: Used to sort the elements in the collection in ascending order (or descending using descending keyword). 
    // Example of 'orderby'
string[] names = { "John", "Alice", "Bob" };
var sortedNames = from name in names
                  orderby name
                  select name;
  • select: Used to determine the shape of the data returned by the query. It specifies what properties or objects should be included in the final result. 
    // Example of 'select' (selecting only names from objects)
var studentNames = from s in studentList
                   select s.Name;
  • contains: Checks whether a specific element exists in a sequence. (Note: Contains is heavily used as an extension method inside the where clause to filter data based on substrings or collection membership). 
    // Example of 'contains'
string[] fruits = { "Apple", "Banana", "Mango" };
var aFruits = from f in fruits
              where f.Contains("a") // true for Banana, Mango
              select f;

⚡ Cheat Sheet

  • Query Syntax: Looks like SQL (from x in list where condition select x).
  • where: Filters data.
  • orderby: Sorts data.
  • select: Shapes/returns data.
  • Contains(): Checks if item or substring exists inside a collection/string.

9. a) What is an event? Explain with suitable program, how event is handled in C#.
b) What is Lambda expression? Explain the concept of generic delegates with suitable program.

Part A: Events in C#

An event is a mechanism in C# that enables a class or object to notify other classes or objects when something of interest happens (e.g., a button click, a process completion). The class that sends (or raises) the event is called the Publisher, and the class that receives (or handles) the event is called the Subscriber.

Events are built on top of delegates. The publisher maintains a delegate, and subscribers add their event-handling methods to this delegate. 

using System;

// 1. Define a delegate for the event
public delegate void Notify();

// Publisher Class
class ProcessBusinessLogic
{
    // 2. Declare the event using the delegate
    public event Notify ProcessCompleted;

    public void StartProcess()
    {
        Console.WriteLine("Process Started...");
        // Some processing happens here...
        
        // 3. Raise the event if there are subscribers
        if (ProcessCompleted != null)
            ProcessCompleted(); 
    }
}

// Subscriber Class
class Program
{
    static void Main()
    {
        ProcessBusinessLogic bl = new ProcessBusinessLogic();
        
        // 4. Subscribe to the event (Attach the event handler)
        bl.ProcessCompleted += bl_ProcessCompleted;
        
        // Start the process
        bl.StartProcess();

        Console.ReadLine();
    }

    // Event Handler Method
    static void bl_ProcessCompleted()
    {
        Console.WriteLine("Process Completed Successfully!");
    }
}

Part B: Lambda Expressions and Generic Delegates

A Lambda Expression is an anonymous function (a function without a name) that you can use to create delegates or expression tree types. It uses the => operator (read as "goes to"). It provides a much shorter, more concise way to write methods.

Generic Delegates are built-in delegates provided by C# (in the System namespace) so you don't have to define custom delegates every time. The three main ones are:

  • Func: Can take up to 16 input parameters and must return a value.
  • Action: Can take up to 16 input parameters but does not return a value (void).
  • Predicate: Takes exactly 1 input parameter and must return a boolean (true/false).
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("--- Generic Delegates with Lambdas ---");

        // 1. Func Delegate: Takes two ints, returns their sum (an int)
        Func<int, int, int> add = (x, y) => x + y;
        Console.WriteLine("Func Result (5 + 10): " + add(5, 10));

        // 2. Action Delegate: Takes a string, returns nothing (void)
        Action<string> printMessage = (msg) => Console.WriteLine("Action Output: " + msg);
        printMessage("Hello from Action!");

        // 3. Predicate Delegate: Takes an int, returns boolean
        Predicate<int> isEven = (num) => num % 2 == 0;
        Console.WriteLine("Predicate Result (Is 4 even?): " + isEven(4));

        Console.ReadLine();
    }
}

⚡ Cheat Sheet

  • Event: A notification mechanism. Publisher raises it, Subscriber handles it via +=.
  • Lambda Expression: (input_parameters) => expression
  • Func: Returns a value. Last type parameter is the return type.
  • Action: Returns void.
  • Predicate: Returns a boolean.
10. How this keyword is used to declare indexer? Write C# program to overload unary (++) and relational (==) operator.

1. Using 'this' Keyword to Declare an Indexer

An indexer allows an object to be indexed such as an array. When you define an indexer for a class, you use the this keyword instead of a normal method name.

The this keyword indicates that the object itself can be accessed using the array access operator [ ]. Behind the scenes, it behaves like a property with get and set accessors, but it takes parameters (like an index). 

// Example syntax of an indexer
public string this[int index]
{
    get { return array[index]; }
    set { array[index] = value; }
}

2. Operator Overloading (++ and ==)

In C#, you can redefine the behavior of operators for custom classes. When overloading the relational operator ==, you must also overload the != operator. Also, it's highly recommended to override Equals() and GetHashCode() when overloading equality operators. 

using System;

class Point
{
    public int X { get; set; }
    public int Y { get; set; }

    public Point(int x, int y)
    {
        X = x;
        Y = y;
    }

    // Overloading Unary Operator (++)
    public static Point operator ++(Point p)
    {
        return new Point(p.X + 1, p.Y + 1);
    }

    // Overloading Relational Operator (==)
    public static bool operator ==(Point p1, Point p2)
    {
        return p1.X == p2.X && p1.Y == p2.Y;
    }

    // Overloading Relational Operator (!=) - Required when overloading ==
    public static bool operator !=(Point p1, Point p2)
    {
        return !(p1 == p2);
    }

    // Required Overrides when overloading equality
    public override bool Equals(object obj)
    {
        if (obj is Point p)
            return this == p;
        return false;
    }

    public override int GetHashCode()
    {
        return X.GetHashCode() ^ Y.GetHashCode();
    }
}

class Program
{
    static void Main()
    {
        Point pt1 = new Point(10, 20);
        Point pt2 = new Point(10, 20);

        Console.WriteLine("--- Relational Operator (==) ---");
        if (pt1 == pt2)
            Console.WriteLine("Points are exactly equal.");

        Console.WriteLine("\n--- Unary Operator (++) ---");
        pt1++; // Calls overloaded ++
        Console.WriteLine("After ++ : pt1.X = " + pt1.X + ", pt1.Y = " + pt1.Y);

        Console.ReadLine();
    }
}

⚡ Cheat Sheet

  • Indexer: Uses this keyword. Makes objects act like arrays: obj[0] = 5;
  • Operator Overloading: Methods must be public static.
  • Paired Operators: If you overload ==, you MUST overload !=. If you overload <, you MUST overload >.
11. What are component of ASP.NET? Explain various validation server control in ASP.NET with suitable program.

1. Components of ASP.NET

ASP.NET is a powerful framework for building web applications. Its core components include:

  • Common Language Runtime (CLR): The execution engine that handles memory management, security, and exception handling for ASP.NET applications.
  • ASP.NET Web Forms / MVC / Core: The different programming models available for building the UI and web logic.
  • Server Controls: Reusable UI components (like buttons, textboxes, grids) that execute on the server but render as standard HTML on the client.
  • ADO.NET: The data access technology used to connect to databases and manage data natively.
  • State Management: Built-in mechanisms like Session, ViewState, and Cookies to maintain state across stateless HTTP requests.

2. Validation Server Controls in ASP.NET

ASP.NET provides Validation Server Controls to easily validate user input. These controls execute both on the client side (using JavaScript for instant feedback) and on the server side (for security). The main validation controls are:

  • RequiredFieldValidator: Ensures the user does not skip a required input field.
  • RangeValidator: Checks if the user's input falls within a specific minimum and maximum range (numbers, dates, etc.).
  • RegularExpressionValidator: Validates input against a specific pattern (e.g., email addresses, phone numbers) using Regex.
  • CompareValidator: Compares the value of one input control to another (e.g., "Password" and "Confirm Password").
  • CustomValidator: Allows you to write your own custom logic (client-side or server-side) for validation.
  • ValidationSummary: Displays a summary list of all validation errors on the page in one place.
