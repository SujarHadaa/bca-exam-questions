2. What is CLR? List and explain the feature of CLR.

Common Language Runtime (CLR) is the basic and Virtual Machine component of the .NET Framework. It acts as the run-time environment that runs the code and manages the execution of .NET programs regardless of the .NET programming language used. It makes the development process easier by providing various services such as memory management, thread management, type-safety, and robustness.

Code that targets the runtime is known as Managed Code, while code that does not target the runtime is known as Unmanaged code.

Features of Common Language Runtime (CLR)

  • Code Access Security: The runtime enforces security by ensuring users can trust executables embedded in a webpage (e.g., to play an animation) while preventing them from accessing personal data, file systems, or networks.
  • Code Robustness: It enforces robustness by implementing a strict type-and-code verification infrastructure known as the Common Type System (CTS), ensuring that all managed code is self-describing.
  • Developer Productivity: It accelerates developer productivity by allowing programmers to write applications in their preferred language while taking full advantage of the runtime, class libraries, and components written by others in different languages.
  • Cross-Era Software Support: While the runtime is built for future software, it is also designed to support software of today and yesterday, easing the migration process for existing applications.
  • Enhanced Performance: The runtime is designed to optimize and enhance the performance of applications through features like Just-In-Time (JIT) compilation.
  • Server-Side Hosting: The runtime can be hosted by high-performance, server-side applications, such as Microsoft SQL Server and Internet Information Services (IIS).
3. What do you mean by value type and reference type? Give an example for each.

In C#, all types are divided mainly into two categories: Value Types and Reference Types. The fundamental difference between them is how they are handled and stored in memory.

Value Types

A Value Type variable directly contains its data (value). When you assign one value type variable to another, a direct copy of the value is made. They are typically stored in a memory area called the Stack. Examples of value types include primitive types (like int, float, char, bool), struct, and enum.

Example:

int a = 10;
int b = a; // A copy of the value 10 is assigned to b

Reference Types

A Reference Type variable does not contain the actual data itself. Instead, it contains a reference (or memory address) to the object that holds the data. They are stored in a memory area called the Heap. When you assign one reference type variable to another, both variables point to the same memory location, meaning changes made through one variable affect the other. Examples of reference types include class, string, array, interface, and delegate.

Example:

class Student { public int age; }
Student s1 = new Student();
s1.age = 20;
Student s2 = s1; // Both s1 and s2 now refer to the same object
s2.age = 25; // s1.age also becomes 25

Key Differences

Feature Value Type Reference Type
Storage Stored in the Stack memory. Stored in the Heap memory.
Data Content Directly holds the actual data. Holds the memory address (reference) to the data.
Assignment Assigning a value type creates an independent copy. Assigning a reference type copies the memory address (both variables point to the same object).
4. What is an interface? How interface is used in multiple inheritance? Explain with example.

What is an Interface?

An interface in C# looks like a class but has no implementation. It contains only the declarations of members like methods, properties, events, and indexers. It acts as a blueprint for a class—it specifies what a class must do, but not how it should do it. The implementation must be provided by the class or struct that inherits the interface.

  • By default, methods declared inside an interface are abstract (they only have a signature and no body).
  • It is used to achieve total abstraction.
  • If a class implements an interface but does not provide bodies for all its functions, the class must be declared abstract.

Interfaces and Multiple Inheritance

Multiple Inheritance refers to the ability of a single class to inherit from more than one parent class simultaneously. While C# does not support multiple inheritance of classes (to prevent complexity and the diamond problem), it fully supports multiple inheritance through interfaces. A class can inherit from one base class and simultaneously implement multiple interfaces.

Example of Multiple Inheritance using Interfaces

The following example demonstrates how a class (BB) inherits from a base class (CA) and implements an interface (IA) to achieve multiple inheritance: 

interface IA
{
    // Interfaces do not contain fields, only declarations
    int CalculateArea();
    int CalculatePerimeter();
}

class CA
{
    public int l, b;
    public void ReadData(int l, int b)
    {
        this.l = l;
        this.b = b;
    }
}

// Class BB achieves multiple inheritance by inheriting from class CA and interface IA
class BB : CA, IA
{
    public int CalculateArea()
    {
        ReadData(10, 5);
        int area = l * b;
        return area;
    }

    public int CalculatePerimeter()
    {
        ReadData(15, 10);
        int peri = 2 * (l + b);
        return peri;
    }
}

// Driver class
class Program
{
    static void Main(string[] args)
    {
        BB obj = new BB();
        Console.WriteLine("Area of Rectangle = " + obj.CalculateArea());
        Console.WriteLine("Perimeter of Rectangle = " + obj.CalculatePerimeter());
    }
}
5. Why namespace is used in C#? How to create and use the namespace in C#? Explain with example.

Why use Namespaces in C#?

A namespace is a domain for type names. Types are typically organized into hierarchical namespaces to make them easier to find and to avoid naming conflicts. By grouping related classes, interfaces, and other types, namespaces keep the codebase organized. For example, the RSA type that handles public-key encryption is neatly organized within the System.Security.Cryptography namespace.

How to create a Namespace

You can create a namespace using the namespace keyword followed by the namespace name, defining types (like classes) within its block. Namespaces can also be nested to create a hierarchy (using dots or nested blocks). 

namespace Outer.Middle.Inner
{
    class Class1 { }
    class Class2 { }
}

How to use a Namespace

You can use the types declared inside a namespace in two ways:
1. By writing the fully qualified name (e.g., Outer.Middle.Inner.Class1).
2. By using the using directive, which imports the namespace and allows you to refer to types without their fully qualified names.

Example

using System;
using Outer.Middle.Inner; // Importing the namespace

namespace ExampleApp
{
    class Test
    {
        static void Main()
        {
            // We can use Class1 directly because we imported its namespace
            Class1 c1 = new Class1();
            
            Console.WriteLine("Namespace imported and class used successfully!");
        }
    }
}
6. What is an Exception? Write a program to generate divide by zero exception and index out of Range exception and also handle these exception.

What is an Exception?

An Exception is a problem or an error that arises during the execution of a program. When an error occurs, C# generates an exception which disrupts the normal flow of the program and terminates it abnormally. To prevent this, we use Exception Handling techniques (like try, catch, and finally blocks) to catch the error and execute alternative clean-up code, allowing the program to continue or terminate gracefully.

Example Program

The following C# program demonstrates how to generate and handle both a DivideByZeroException and an IndexOutOfRangeException using multiple catch blocks: 

using System;

class ExceptionDemo
{
    static void Main(string[] args)
    {
        try
        {
            // 1. Generating DivideByZeroException
            int numerator = 10;
            int denominator = 0;
            Console.WriteLine("Attempting to divide by zero...");
            int result = numerator / denominator; // This line throws an exception

            // 2. Generating IndexOutOfRangeException (This code will be skipped because of the previous error)
            int[] arr = new int[3];
            Console.WriteLine("Attempting to access invalid index...");
            arr[5] = 100; // This line would throw an exception if reached
        }
        catch (DivideByZeroException ex)
        {
            // Handling Divide By Zero Error
            Console.WriteLine("Error: Cannot divide a number by zero!");
            Console.WriteLine("Exception details: " + ex.Message);
        }
        catch (IndexOutOfRangeException ex)
        {
            // Handling Index Out Of Range Error
            Console.WriteLine("Error: Array index is out of bounds!");
            Console.WriteLine("Exception details: " + ex.Message);
        }
        catch (Exception ex)
        {
            // General fallback for any other exceptions
            Console.WriteLine("An unexpected error occurred: " + ex.Message);
        }
        finally
        {
            // Finally block always executes
            Console.WriteLine("Program execution completed.");
        }
    }
}
7. What is a Query expression? Write a C# program to select Students who are lived in Kirtipur and studied in Patan Multiple Campus using LINQ.

What is a Query Expression?

A Query Expression is a query written in query syntax that is similar to SQL or XQuery. In C# LINQ (Language Integrated Query), a query expression is used to extract, filter, and sort data from any enumerable data source (like arrays, collections, databases, or XML). It typically starts with a from clause and ends with a select or group clause.

C# LINQ Program Example

The following program demonstrates how to create a list of students and use a LINQ query expression to filter out the students living in "Kirtipur" and studying at "Patan Multiple Campus". 

using System;
using System.Collections.Generic;
using System.Linq;

class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string Address { get; set; }
    public string Campus { get; set; }
}

class Program
{
    static void Main(string[] args)
    {
        // 1. Data Source: Creating a list of students
        List<Student> studentList = new List<Student>()
        {
            new Student() { Id = 1, Name = "Ram", Address = "Kirtipur", Campus = "Patan Multiple Campus" },
            new Student() { Id = 2, Name = "Sita", Address = "Baneshwor", Campus = "Patan Multiple Campus" },
            new Student() { Id = 3, Name = "Hari", Address = "Kirtipur", Campus = "Mechi Multiple Campus" },
            new Student() { Id = 4, Name = "Gita", Address = "Kirtipur", Campus = "Patan Multiple Campus" }
        };

        // 2. Query Expression: Filtering students
        var filteredStudents = from s in studentList
                               where s.Address == "Kirtipur" && s.Campus == "Patan Multiple Campus"
                               select s;

        // 3. Query Execution: Displaying results
        Console.WriteLine("Students from Kirtipur studying in Patan Multiple Campus:\n");
        foreach (var student in filteredStudents)
        {
            Console.WriteLine("ID: " + student.Id + ", Name: " + student.Name);
        }
        
        Console.ReadLine();
    }
}
8. What do you mean by static class and constructor? Explain with suitable example.

Static Class

A Static Class is a class that cannot be instantiated (you cannot create an object of it using the new keyword). The sole purpose of a static class is to contain static members (methods, properties, etc.). The static keyword is used before the class keyword to declare a static class. Since it has no instances, its members are accessed directly using the class name.

Static Constructor

A Static Constructor is used to initialize any static data or to perform a particular action that needs to be performed only once. It is invoked automatically before the first instance is created or any static members are referenced.

  • A static constructor does not take any access modifiers or parameters.
  • A class can only have one static constructor.
  • It is called automatically by the CLR, meaning the developer has no control over its execution order.

Example of Static Class and Constructor

using System;

static class College
{
    public static string Name;

    // Static Constructor
    static College()
    {
        Name = "Mechi Multiple Campus";
        Console.WriteLine("Static Constructor called!");
    }

    public static void Display()
    {
        Console.WriteLine("College Name: " + Name);
    }
}

class Program
{
    static void Main()
    {
        // Accessing static member directly. 
        // This will automatically invoke the static constructor first.
        College.Display();
    }
}

9. What do you mean by Operator Overloading? Write a C# program to demonstrate binary and relational operator overloading.

Operator Overloading

Operator Overloading is a concept that gives the ability to use the same operator to perform various custom operations on user-defined data types (like classes and structs). It provides additional capabilities to C# operators, enabling developers to write intuitive and readable code. For example, you can overload the + operator to add two objects of a custom class together. An operator is overloaded by defining a static function using the operator keyword.

Program Demonstrating Binary and Relational Operator Overloading

The following program overloads the binary operator + and the relational operators == and != for a custom Box class: 

using System;

class Box
{
    public int Length;
    public int Breadth;

    public Box(int l, int b)
    {
        Length = l;
        Breadth = b;
    }

    // 1. Overloading Binary Operator (+)
    public static Box operator +(Box b1, Box b2)
    {
        Box box = new Box(0, 0);
        box.Length = b1.Length + b2.Length;
        box.Breadth = b1.Breadth + b2.Breadth;
        return box;
    }

    // 2. Overloading Relational Operator (==)
    // Note: Relational operators must be overloaded in pairs (e.g., == and !=)
    public static bool operator ==(Box b1, Box b2)
    {
        return (b1.Length == b2.Length) && (b1.Breadth == b2.Breadth);
    }

    // 3. Overloading Relational Operator (!=)
    public static bool operator !=(Box b1, Box b2)
    {
        return !(b1 == b2);
    }
}

class Program
{
    static void Main()
    {
        Box box1 = new Box(10, 20);
        Box box2 = new Box(10, 20);
        Box box3 = new Box(5, 5);

        // Binary Operator Overloading Demo
        Box box4 = box1 + box3;
        Console.WriteLine("Box 4 Dimensions: Length = " + box4.Length + ", Breadth = " + box4.Breadth);

        // Relational Operator Overloading Demo
        if (box1 == box2)
            Console.WriteLine("Box1 is equal to Box2.");
        else
            Console.WriteLine("Box1 is NOT equal to Box2.");

        if (box1 != box3)
            Console.WriteLine("Box1 is NOT equal to Box3.");
        else
            Console.WriteLine("Box1 is equal to Box3.");

        Console.ReadLine();
    }
}
10. Explain difference between ExecuteReader and ExecuteNonQuery. Provided that a mysql database named "Company" with table named "Product" with following columns (ProductId as int, ProductName as varchar(20), and UnitPrice as float). Write a C# program to connect to the database and display product records that have UnitPrice is greater than RS. 1000 and update the product record (such as Unit Price 4500) from ProductId whose ProductId=50.

Difference between ExecuteReader and ExecuteNonQuery

ExecuteReader ExecuteNonQuery
Used when a SQL statement returns data (e.g., SELECT queries). Used when a SQL statement does not return data (e.g., INSERT, UPDATE, DELETE).
Returns a DataReader object that can read records row by row. Returns an integer representing the number of rows affected by the query.
Best for fetching a list of records from the database. Best for making changes to the database structure or data.

C# Program for MySQL Database

The following program connects to the "Company" MySQL database, fetches products with a price > 1000 using ExecuteReader, and updates a product using ExecuteNonQuery. 

using System;
using MySql.Data.MySqlClient; // MySQL namespace

class Program
{
    static void Main(string[] args)
    {
        // 1. Database Connection String
        string connString = "server=localhost;database=Company;uid=root;password=yourpassword;";

        // 2. Establishing Connection
        using (MySqlConnection conn = new MySqlConnection(connString))
        {
            try
            {
                conn.Open();
                Console.WriteLine("Connected to MySQL Database successfully!");

                // 3. Displaying records using ExecuteReader (SELECT query)
                string selectQuery = "SELECT * FROM Product WHERE UnitPrice > 1000";
                MySqlCommand cmdSelect = new MySqlCommand(selectQuery, conn);

                Console.WriteLine("\nProducts with Unit Price > 1000:");
                MySqlDataReader reader = cmdSelect.ExecuteReader();
                while (reader.Read())
                {
                    Console.WriteLine("ID: " + reader["ProductId"] + 
                                      ", Name: " + reader["ProductName"] + 
                                      ", Price: RS. " + reader["UnitPrice"]);
                }
                reader.Close(); // Close the reader before executing another command

                // 4. Updating record using ExecuteNonQuery (UPDATE query)
                string updateQuery = "UPDATE Product SET UnitPrice = 4500 WHERE ProductId = 50";
                MySqlCommand cmdUpdate = new MySqlCommand(updateQuery, conn);
                
                int rowsAffected = cmdUpdate.ExecuteNonQuery();
                if (rowsAffected > 0)
                {
                    Console.WriteLine("\nProduct updated successfully! Rows affected: " + rowsAffected);
                }
                else
                {
                    Console.WriteLine("\nProduct with ID 50 not found.");
                }
            }
            catch (Exception ex)
            {
                Console.WriteLine("Error: " + ex.Message);
            }
        }
    }
}
11. What is web server control in ASP.NET? List five web server control used in ASP.NET. Write a program to create student registration form in one ASP.NET page and display the filled data within another page.

What is a Web Server Control?

Web Server Controls are special ASP.NET tags that run on the server and are converted into standard HTML before being sent to the user's browser. They provide built-in features such as state management (remembering data between postbacks), rich event handling, and automatic browser detection. They are declared using the asp: prefix and must always include the runat="server" attribute.

Five Web Server Controls in ASP.NET

  • Label (<asp:Label>) - Used to display text on the web page.
  • TextBox (<asp:TextBox>) - Used to accept user input (text).
  • Button (<asp:Button>) - Used to submit a form or trigger an action when clicked.
  • DropDownList (<asp:DropDownList>) - Used to allow the user to select an item from a drop-down list.
  • CheckBox (<asp:CheckBox>) - Used to represent a true/false or yes/no state.

ASP.NET Program: Student Registration Form

The following program uses two pages. The first page (Registration.aspx) takes the student details and uses the PostBackUrl property of the button to send the data directly to the second page (Display.aspx).

1. Registration.aspx (The Form Page)

<%@ Page Language="C#" %>
<!DOCTYPE html>
<html>
<body>
    <form id="form1" runat="server">
        <h3>Student Registration</h3>
        
        <asp:Label ID="lblName" runat="server" Text="Name: "></asp:Label>
        <asp:TextBox ID="txtName" runat="server"></asp:TextBox>
        <br /><br />
        
        <asp:Label ID="lblEmail" runat="server" Text="Email: "></asp:Label>
        <asp:TextBox ID="txtEmail" runat="server"></asp:TextBox>
        <br /><br />
        
        <!-- PostBackUrl sends data to Display.aspx -->
        <asp:Button ID="btnSubmit" runat="server" Text="Register" PostBackUrl="Display.aspx" />
    </form>
</body>
</html>

2. Display.aspx (The Display Page)

<%@ Page Language="C#" %>
<!DOCTYPE html>
<html>
<body>
    <form id="form2" runat="server">
        <h3>Registration Details</h3>
        <% 
            // Retrieving data from the previous page using Request.Form
            string studentName = Request.Form["txtName"];
            string studentEmail = Request.Form["txtEmail"];
            
            // Displaying the submitted data
            Response.Write("Student Name: " + studentName + "<br />");
            Response.Write("Student Email: " + studentEmail);
        %>
    </form>
</body>
</html>
