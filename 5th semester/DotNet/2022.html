2. What do you mean by "safe" programming language? Explain any four applied technologies in DotNet framework.

1. Safe Programming Language (Type Safety)

A "safe" programming language (like C#) refers to a language that guarantees Type Safety. This means that instances of types can interact only through protocols they explicitly define, which ensures the internal consistency of each type. For example, C# strictly prevents you from interacting with a string type as though it were an int (integer).

More specifically, C# enforces static typing, meaning that type safety is strictly checked by the compiler at compile-time before the program even runs. This eliminates a huge class of errors, making the software much easier to manage, more predictable, and highly robust.

2. Four Applied Technologies in the .NET Framework

The .NET Framework offers various applied technologies for different development scenarios. Four major ones include:

  • 1. ASP.NET: A mature technology for creating web applications that run on a server (IIS) and are accessed via any web browser. It features zero deployment at the client side and allows easy updates.
  • 2. Windows Presentation Foundation (WPF): Introduced in Framework 3.0, it is used for writing rich-client desktop applications. It offers sophisticated graphics (3D rendering, true transparency) using DirectX and allows user interfaces to be designed cleanly using XAML files.
  • 3. Windows Forms: An older, relatively simple rich-client API for building traditional Windows desktop applications. While it lacks some advanced graphics features of WPF, it remains relevant for legacy systems and quick application development.
  • 4. Xamarin: A technology that enables developers to write cross-platform mobile applications in C# that target iOS, Android, and Windows Phone from a single shared codebase.

⚡ Cheat Sheet

  • Safe Language: Means "Type Safety". A string cannot be accidentally used as an integer. Errors are caught early at compile-time.
  • ASP.NET: For building Websites and Web Apps.
  • WPF: For building modern Desktop Apps with rich graphics (XAML).
  • Windows Forms: For building traditional/simple Desktop Apps.
  • Xamarin: For building Mobile Apps (iOS/Android) using C#.
3. List any five canonical keywords in "C#". Write a C# program to initialize and display array element with sum of each power.

1. Five Canonical Keywords in C#

In C#, keywords are reserved words that hold a special predefined meaning to the compiler. They cannot be used as identifiers (like variable or class names) unless prefixed with the @ symbol. Five standard (canonical) reserved keywords in C# are:

  • 1. public: An access modifier that makes a class or member accessible from anywhere.
  • 2. class: Used to declare a new class (a blueprint for creating objects).
  • 3. void: Specifies that a method does not return a value.
  • 4. static: Declares a member that belongs to the type itself rather than a specific instance/object.
  • 5. int: A predefined data type used to store 32-bit signed integers.

2. Array Initialization and Sum of Powers Code Example

using System;

public class ArrayPowerSum
{
    public static void Main(string[] args)
    {
        // Initialize the array
        int[] numbers = { 1, 2, 3, 4 };
        double totalSum = 0;
        
        Console.WriteLine("Array Elements and their Squares:");
        
        // Loop through the array
        for (int i = 0; i < numbers.Length; i++)
        {
            double power = Math.Pow(numbers[i], 2); // Calculating power of 2
            totalSum += power;
            Console.WriteLine($"Element: {numbers[i]}, Square: {power}");
        }
        
        Console.WriteLine($"\nTotal Sum of powers: {totalSum}");
    }
}

⚡ Cheat Sheet

  • Keywords: Reserved words with special meaning (e.g., public, class, void, int, static).
  • Math.Pow(x, y): C# method used to calculate powers. Evaluates x raised to the power y.
  • Array Iteration: for (int i = 0; i < arr.Length; i++) is the standard loop to read all array elements.
4. What is string interpolation? How passing argument by value is differing from passing argument by reference? Explain with program.

1. String Interpolation

String Interpolation is a feature in C# that allows you to easily format strings by embedding variables and expressions directly inside a string literal. This is done by prefixing the string with a $ character and wrapping variables in curly braces {}. It is a cleaner, more readable alternative to traditional string concatenation using +.

2. Passing by Value vs. Passing by Reference

  • Passing by Value: This is the default mechanism in C#. When an argument is passed by value, a copy of the variable's value is created and passed to the method. Any modifications made to the parameter inside the method do not affect the original variable because they exist in different memory locations.
  • Passing by Reference: When using the ref keyword, you pass the actual memory address of the variable rather than a copy. As a result, any changes made to the parameter within the method will directly update the original variable's value.

3. Code Example

using System;

public class PassMethodDemo
{
    // 1. Pass by Value Method
    public static void PassByValue(int x)
    {
        x = x + 10; // Modifies only the local copy
        Console.WriteLine($"Inside PassByValue: {x}"); // Using String Interpolation
    }

    // 2. Pass by Reference Method
    public static void PassByReference(ref int y)
    {
        y = y + 10; // Modifies the actual original variable
        Console.WriteLine($"Inside PassByReference: {y}"); // Using String Interpolation
    }

    public static void Main(string[] args)
    {
        int num1 = 5;
        Console.WriteLine($"Before PassByValue: {num1}");
        PassByValue(num1);
        Console.WriteLine($"After PassByValue: {num1}\n"); // Original remains 5

        int num2 = 5;
        Console.WriteLine($"Before PassByReference: {num2}");
        PassByReference(ref num2);
        Console.WriteLine($"After PassByReference: {num2}"); // Original becomes 15
    }
}

⚡ Cheat Sheet

  • String Interpolation: Uses $ before quotes to insert variables via {} (e.g. $"Value: {x}").
  • Pass by Value: Default in C#. Creates a copy of data. Original is safe.
  • Pass by Reference: Requires ref. Passes memory address. Original can be changed.
5. What are sealed classes in C#? Write a C# program to illustrate the concept of abstract class and abstract method.

1. Sealed Classes

In C#, Sealed Classes are used to restrict the inheritance feature of Object-Oriented Programming. Once a class is defined using the sealed keyword, it cannot be inherited by any other class.

This is often used for security reasons or to prevent unintended modification of a class's behavior. If another class tries to derive from a sealed class, the C# compiler will throw an error. (Note: Structs are implicitly sealed in C#).

2. Abstract Class and Abstract Method Program

An Abstract Class is a class that cannot be instantiated directly and is meant only to be inherited. An Abstract Method is a method declared without a body inside an abstract class, forcing the derived class to provide the actual implementation using the override keyword. 

using System;

// Abstract Class
public abstract class Shape 
{ 
    // Abstract Method (No body)
    public abstract void Draw(); 
} 

// Derived Concrete Class
public class Rectangle : Shape 
{ 
    // Overriding the abstract method
    public override void Draw() 
    { 
        Console.WriteLine("Drawing a Rectangle..."); 
    } 
}

public class Program 
{ 
    public static void Main() 
    { 
        // Cannot instantiate Shape directly (e.g. new Shape() is illegal)
        // Instead, we instantiate the derived class Rectangle.
        Shape myShape = new Rectangle(); 
        myShape.Draw(); // Outputs: Drawing a Rectangle...
    } 
}

⚡ Cheat Sheet

  • Sealed Class: Uses the sealed keyword. Cannot be inherited by any other class.
  • Abstract Class: Uses the abstract keyword. Cannot be instantiated directly using new. Must be inherited.
  • Abstract Method: Has no body. Must be overridden (override) in the child class. Used to achieve abstraction.
6. What is virtual method? Write a program to illustrate multi-level inheritance with virtual method.

1. Virtual Method

A Virtual Method is a method in a base class that can be redefined (overridden) in its derived classes. It provides a default implementation that can either be used as-is by the derived class, or replaced with a more specific implementation.

To declare a virtual method, you use the virtual keyword in the base class. To modify it in the derived class, you use the override keyword. This is the foundation of Runtime Polymorphism (Dynamic Binding) in C#.

2. Multi-Level Inheritance with Virtual Methods Program

Multi-level Inheritance occurs when a class is derived from another derived class (e.g., Grandparent → Parent → Child). Below is a program demonstrating this concept using a virtual method. 

using System;

// Level 1: Base Class
public class Grandparent
{
    // Virtual method allowing overriding
    public virtual void Display()
    {
        Console.WriteLine("Display from Grandparent");
    }
}

// Level 2: Intermediate Derived Class
public class Parent : Grandparent
{
    // Overriding the virtual method
    public override void Display()
    {
        Console.WriteLine("Display from Parent");
    }
}

// Level 3: Final Derived Class (Multi-level)
public class Child : Parent
{
    // Overriding the Parent's method
    public override void Display()
    {
        Console.WriteLine("Display from Child");
    }
}

public class Program
{
    public static void Main()
    {
        // Demonstrating dynamic polymorphism
        Grandparent gp = new Grandparent();
        gp.Display(); // Outputs: Display from Grandparent

        Grandparent p = new Parent();
        p.Display();  // Outputs: Display from Parent

        Grandparent c = new Child();
        c.Display();  // Outputs: Display from Child
    }
}

⚡ Cheat Sheet

  • Virtual Method: Uses virtual. Provides a base implementation that can be overridden.
  • Override: Uses override. Replaces the base implementation in the child class.
  • Multi-Level Inheritance: A chain of inheritance where a derived class serves as a base class for another (e.g., A → B → C).
7. What is different between index and properties? Explain the concept of generic class in C# with a suitable program.

1. Differences between Indexers and Properties

Both Indexers and Properties allow data encapsulation, but they are used in different ways. A property allows access to a single piece of data (a field), whereas an indexer allows a class or struct to be indexed exactly like an array.

Indexers Properties
Created using the this keyword. Do not require the this keyword.
Identified by their signature (parameters). Identified by their names.
Accessed using array-like brackets [ ]. Accessed by their simple names (e.g. obj.Name).
Must be an instance member; cannot be static. Can be both instance and static members.
A get accessor has the same formal parameter list as the indexer. A get accessor has no parameters.

2. Generic Class Concept and Program

A Generic Class in C# allows you to define a class with placeholders (type parameters, usually <T>) for the data types it will store or use. This makes the class highly reusable, type-safe, and performant because you can use the exact same class logic with different data types without boxing or unboxing. 

using System;

// A Generic Class defined with type parameter <T>
public class GenericBox<T>
{
    private T item;

    // Method that accepts a generic type T
    public void Add(T newItem)
    {
        item = newItem;
    }

    // Method that returns a generic type T
    public T GetItem()
    {
        return item;
    }
}

public class Program
{
    public static void Main()
    {
        // Using the generic class to store an Integer
        GenericBox<int> intBox = new GenericBox<int>();
        intBox.Add(100);
        Console.WriteLine($"Integer Box contains: {intBox.GetItem()}");

        // Using the exact same generic class to store a String
        GenericBox<string> stringBox = new GenericBox<string>();
        stringBox.Add("Hello Generics!");
        Console.WriteLine($"String Box contains: {stringBox.GetItem()}");
    }
}

⚡ Cheat Sheet

  • Indexer: Uses this[int index]. Acts like an array. Cannot be static.
  • Property: Named wrapper over fields. Uses get and set. Can be static.
  • Generic Class: Defined with <T>. Allows class reuse with multiple data types, ensuring type safety.

9. a) System Exception and Withdrawal Program

System Exception: In C#, a SystemException is the base class for all predefined runtime exceptions thrown by the Common Language Runtime (CLR). Examples include IndexOutOfRangeException, NullReferenceException, and DivideByZeroException. It indicates errors that occur during application execution, generally related to system failures or invalid operations.

Below is the program that reads balance and withdrawal amount, checks validity, and throws an ApplicationException if the withdrawal amount exceeds the balance: 

using System;

public class BankWithdrawal
{
    public static void Main()
    {
        try
        {
            Console.Write("Enter Account Balance: ");
            double balance = Convert.ToDouble(Console.ReadLine());

            Console.Write("Enter Withdrawal Amount: ");
            double withdraw = Convert.ToDouble(Console.ReadLine());

            if (withdraw > balance)
            {
                // Throw ApplicationException when custom business logic fails
                throw new ApplicationException("Insufficient Balance! Withdrawal amount exceeds current balance.");
            }
            else
            {
                balance -= withdraw;
                Console.WriteLine($"Withdrawal Successful! Remaining Balance: {balance}");
            }
        }
        catch (ApplicationException ex)
        {
            Console.WriteLine($"Error: {ex.Message}");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"Invalid Input: {ex.Message}");
        }
    }
}

9. b) LINQ Operators and Aggregate Salary Program

LINQ (Language Integrated Query) Standard Operators: These are methods that form the LINQ pattern, allowing you to query data. Five common LINQ operators are:

  • 1. Where: Filters a sequence of values based on a predicate.
  • 2. Select: Projects each element of a sequence into a new form.
  • 3. OrderBy / OrderByDescending: Sorts elements in ascending or descending order.
  • 4. Sum: Computes the sum of a sequence of numeric values.
  • 5. GroupBy: Groups elements that share a common attribute.

Below is the program using LINQ to compute aggregate salary and display records in descending order: 

using System;
using System.Linq;
using System.Collections.Generic;

public class Employee
{
    public string Name { get; set; }
    public double Salary { get; set; }
}

public class LinqEmployeeDemo
{
    public static void Main()
    {
        // Create a list of 5 employees
        List<Employee> empList = new List<Employee>()
        {
            new Employee { Name = "Alice", Salary = 45000 },
            new Employee { Name = "Bob", Salary = 62000 },
            new Employee { Name = "Charlie", Salary = 38000 },
            new Employee { Name = "David", Salary = 75000 },
            new Employee { Name = "Eve", Salary = 50000 }
        };

        // 1. Compute Aggregate (Sum) Salary
        double totalSalary = empList.Sum(e => e.Salary);
        Console.WriteLine($"Total Aggregate Salary: {totalSalary}\n");

        // 2. Sort by Salary in Descending Order
        var sortedEmployees = empList.OrderByDescending(e => e.Salary);

        Console.WriteLine("Employees sorted by Salary (Descending):");
        foreach (var emp in sortedEmployees)
        {
            Console.WriteLine($"Name: {emp.Name}, Salary: {emp.Salary}");
        }
    }
}

⚡ Cheat Sheet

  • SystemException vs ApplicationException: SystemException is for CLR errors. ApplicationException is for custom application/business rule errors.
  • Throwing Exception: Use throw new ExceptionType("Message");.
  • LINQ Aggregate: Use .Sum(x => x.Property) to get total.
  • LINQ Sorting: Use .OrderByDescending(x => x.Property).
10. What are different type of delegate in C#? Write a C# Program to create a class 'Time' which represents time. The class should have three fields for hours, minutes and seconds. It should have constructor to initialize hours, minutes and seconds and method Time() to print current time. overload following operators a) +(Add two time objects of 24 hours clock) b) Compare two time objects

1. Types of Delegates in C#

A Delegate in C# is a type-safe function pointer. The different types of delegates are:

  • Singlecast Delegate: A delegate that points to a single method at a time.
  • Multicast Delegate: A delegate that can point to multiple methods at once. When invoked, all assigned methods are called in sequence. Used extensively in events.
  • Generic Delegates (Built-in): Pre-defined delegates introduced to avoid declaring custom delegates for common scenarios. They are:
    • Func<>: Returns a value.
    • Action<>: Does not return a value (void).
    • Predicate<>: Always returns a boolean.

2. Time Class and Operator Overloading Program

In C#, you cannot have a method with the exact same name as the class (it would be treated as a constructor). For the method meant to print time, we'll name it DisplayTime(). Operator overloading uses the operator keyword and must be public static. For comparison operators, if you overload ==, you must also overload !=. 

using System;

public class Time
{
    public int Hours;
    public int Minutes;
    public int Seconds;

    // Constructor to initialize fields
    public Time(int h, int m, int s)
    {
        Hours = h;
        Minutes = m;
        Seconds = s;
    }

    // Method to print current time
    public void DisplayTime()
    {
        Console.WriteLine($"{(Hours):D2}:{(Minutes):D2}:{(Seconds):D2}");
    }

    // a) Overload '+' operator for 24-hour clock
    public static Time operator +(Time t1, Time t2)
    {
        int totalSeconds = t1.Seconds + t2.Seconds;
        int s = totalSeconds % 60;
        
        int totalMinutes = t1.Minutes + t2.Minutes + (totalSeconds / 60);
        int m = totalMinutes % 60;
        
        int totalHours = t1.Hours + t2.Hours + (totalMinutes / 60);
        int h = totalHours % 24; // 24 hour clock logic

        return new Time(h, m, s);
    }

    // b) Overload '==' and '!=' operators to compare objects
    public static bool operator ==(Time t1, Time t2)
    {
        if (ReferenceEquals(t1, t2)) return true;
        if (ReferenceEquals(t1, null) || ReferenceEquals(t2, null)) return false;

        return (t1.Hours == t2.Hours) && (t1.Minutes == t2.Minutes) && (t1.Seconds == t2.Seconds);
    }

    public static bool operator !=(Time t1, Time t2)
    {
        return !(t1 == t2);
    }

    // Best practice when overloading ==
    public override bool Equals(object obj)
    {
        if (obj is Time) return this == (Time)obj;
        return false;
    }

    public override int GetHashCode()
    {
        return (Hours, Minutes, Seconds).GetHashCode();
    }
}

public class Program
{
    public static void Main()
    {
        Time t1 = new Time(10, 45, 30);
        Time t2 = new Time(15, 20, 45);

        Console.Write("Time 1: "); t1.DisplayTime();
        Console.Write("Time 2: "); t2.DisplayTime();

        // Using overloaded + operator
        Time t3 = t1 + t2;
        Console.Write("Sum (t1 + t2): "); t3.DisplayTime();

        // Using overloaded == operator
        if (t1 == t2)
            Console.WriteLine("Time 1 and Time 2 are equal.");
        else
            Console.WriteLine("Time 1 and Time 2 are NOT equal.");
    }
}

⚡ Cheat Sheet

  • Delegate Types: Singlecast, Multicast, Generic (Func, Action, Predicate).
  • Operator Overloading Syntax: public static ReturnType operator +(Type t1, Type t2).
  • Comparison Operators: Must be overloaded in pairs (e.g., == and !=). Remember to override Equals() and GetHashCode().
11. What is difference between executeScalar() and executeReader() method? Write a C# program to connect database Bank and insert 5 customer record (Account_no, Name, Address, Balance) in Customer table and Display the customer record whose balance is greater than 5000.

1. ExecuteScalar() vs ExecuteReader()

In ADO.NET, both methods are used to execute queries against a database using a SqlCommand object, but they are used for different purposes:

ExecuteScalar() ExecuteReader()
Returns a single value (the first column of the first row in the result set). Returns a SqlDataReader object that can read multiple rows and columns.
Used mainly for aggregate functions like COUNT(), SUM(), MAX(), MIN(). Used for retrieving large sets of data, typically with a SELECT * FROM Table statement.
Fast and lightweight, as it ignores extra rows and columns. Forward-only, read-only stream of rows. It requires a while(reader.Read()) loop to process data.

2. ADO.NET Database Connection Program

Below is a complete C# program using System.Data.SqlClient to connect to a database named "Bank", insert 5 records using a loop and parameterized queries, and then read the records back using ExecuteReader() to filter those with a balance over 5000. 

using System;
using System.Data.SqlClient;

public class BankDatabase
{
    public static void Main()
    {
        // Connection string for connecting to the "Bank" database
        string connString = @"Data Source=YourServerName;Initial Catalog=Bank;Integrated Security=True";

        using (SqlConnection conn = new SqlConnection(connString))
        {
            try
            {
                conn.Open();
                
                // --- 1. INSERT 5 RECORDS ---
                string insertQuery = "INSERT INTO Customer (Account_no, Name, Address, Balance) VALUES (@acc, @name, @addr, @bal)";

                for (int i = 1; i <= 5; i++)
                {
                    using (SqlCommand insertCmd = new SqlCommand(insertQuery, conn))
                    {
                        // Using parameters prevents SQL Injection
                        insertCmd.Parameters.AddWithValue("@acc", 1000 + i);
                        insertCmd.Parameters.AddWithValue("@name", "Customer_" + i);
                        insertCmd.Parameters.AddWithValue("@addr", "City_" + i);
                        insertCmd.Parameters.AddWithValue("@bal", i * 2000); // Balances will be 2000, 4000, 6000, 8000, 10000

                        insertCmd.ExecuteNonQuery();
                    }
                }
                Console.WriteLine("5 Customer Records Inserted Successfully!\n");


                // --- 2. DISPLAY RECORDS WITH BALANCE > 5000 ---
                Console.WriteLine("--- Customers with Balance > 5000 ---");
                string selectQuery = "SELECT Account_no, Name, Address, Balance FROM Customer WHERE Balance > 5000";

                using (SqlCommand selectCmd = new SqlCommand(selectQuery, conn))
                {
                    // Use ExecuteReader to read multiple rows
                    using (SqlDataReader reader = selectCmd.ExecuteReader())
                    {
                        while (reader.Read())
                        {
                            Console.WriteLine($"Acc No: {reader["Account_no"]}, Name: {reader["Name"]}, Balance: {reader["Balance"]}");
                        }
                    }
                }
            }
            catch (Exception ex)
            {
                Console.WriteLine("Database Error: " + ex.Message);
            }
        }
    }
}

⚡ Cheat Sheet

  • ExecuteScalar(): Used for single-value returns (e.g., COUNT(*)).
  • ExecuteReader(): Used for fetching multiple rows. Requires a while (reader.Read()) loop.
  • ExecuteNonQuery(): Used for INSERT, UPDATE, and DELETE.
  • Parameterized Queries: Always use .AddWithValue("@param", value) to prevent SQL Injection.
