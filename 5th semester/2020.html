1. Differentiate Object Oriented Programming and Object Based Programming. Explain some of the major features of C# language.

Part A: Object-Oriented vs. Object-Based Programming

While both paradigms utilize objects to encapsulate data and behavior, they differ significantly in their support for advanced relationship concepts.

Feature Object-Oriented Programming (OOP) Object-Based Programming (OBP)
Definition A programming paradigm that strictly follows all core concepts of objects, classes, and their relationships. A programming paradigm that uses objects and encapsulates state/behavior, but does not fully support all OOP pillars.
Inheritance Supported. Classes can inherit properties and methods from parent classes, promoting code reusability. Not Supported. Objects cannot inherently derive features from other objects or classes.
Polymorphism Supported. Allows processing objects differently based on their data type or class (e.g., method overloading/overriding). Not Supported. Functions cannot easily be overridden or dynamically bound.
Examples C#, Java, C++ Visual Basic (early versions), VBScript, JavaScript (historically)

Part B: Major Features of C# Language

C# is a modern, general-purpose programming language developed by Microsoft. As per the .NET framework guidelines, its distinctive features include:

  • Pure Object-Oriented: C# is a rich implementation of the object-orientation paradigm. It fully supports encapsulation, abstraction, inheritance, and polymorphism. All methods and variables (even the Main entry point) must be encapsulated within a class.
  • Unified Type System: The fundamental building block is an encapsulated unit called a type. C# uses a unified type system where all types (primitive or complex business objects) ultimately share a common base type (System.Object), meaning they share core functionalities like the ToString() method.
  • Strict Type Safety: C# is primarily a type-safe language enforcing internal consistency. It supports static typing (catching errors at compile-time) and prevents implicit conversions that result in data loss, making large programs robust and predictable.
  • Automatic Memory Management: C# relies on the Common Language Runtime (CLR) for memory management. The CLR's Garbage Collector automatically reclaims memory for objects that are no longer referenced, eliminating incorrect pointer issues seen in languages like C++.
  • Cross-Platform Support: While historically tied to Windows, modern C# (via .NET Core and Xamarin) supports cross-platform development across Windows, Linux, macOS, iOS, and Android.
  • Component Oriented: It is designed for improving productivity in the development of Web applications and distributed components.
2. Explain overview of Microsoft .NET framework and its components in detail.

Overview of Microsoft .NET Framework

.NET is a programming infrastructure created by Microsoft for building, deploying, and running applications and services (such as desktop applications and Web services). It sits between application programs and the operating system.

It provides an object-oriented environment and ensures safe execution of code by performing required runtime validations (e.g., preventing array bounds violations or accessing another program's memory), making the entire environment robust.

.NET Framework Architecture

Figure: Hierarchy of .NET Framework Components

Key Components in Detail

  • Common Language Runtime (CLR): The core execution engine and virtual machine component of the .NET Framework. It manages the execution of .NET programs and provides essential services like memory management, type-safety, exception handling, and thread management. Code that targets the CLR is known as Managed Code.
  • Framework Class Library (FCL): A comprehensive, object-oriented collection of reusable types (classes, interfaces) that can be integrated with the CLR. It is used to develop a wide range of applications from traditional command-line apps to ASP.NET web applications.
  • Common Language Infrastructure (CLI): The underlying infrastructure and specification that allows for language-agnostic development, upon which the CTS and CLS are built.
  • Common Type System (CTS): A strict type-and-code-verification infrastructure. It describes how types are defined and used, ensuring that all managed code is self-describing and enabling seamless cross-language integration.
  • Common Language Specification (CLS): A subset of the CTS. It defines a baseline of rules and standards that all .NET languages must follow to ensure interoperability (e.g., allowing C# code to seamlessly interact with VB.NET code).
  • Garbage Collector (GC): An automatic memory manager executing as part of the CLR. It reclaims memory for objects that are no longer referenced, freeing programmers from explicitly deallocating memory and preventing memory leaks.
  • Just-In-Time Compiler (JIT): A compiler within the CLR that converts Intermediate Language (IL) into native machine code (such as x86 or x64) just prior to execution, significantly enhancing execution speed.
4. What do you mean by property in C# language? How it is different from method? Compare automatic property with other types of property with suitable example.

Part A: What is a Property in C#?

A property is a named member of a class, struct, or interface that looks like a field from the outside but contains logic (like methods) internally. It encapsulates a piece of an object's state (e.g., a button's color, a label's text) using special methods called accessors — get (read) and set (write). Properties provide controlled access to private fields, following encapsulation principles. 

class Student
{
    private string name;  // backing field

    public string Name    // property
    {
        get { return name; }
        set { name = value; }
    }
}

Part B: How is Property Different from Method?

Feature Property Method
Purpose Encapsulates state (data) Performs an action (behavior)
Syntax Accessed like a field (obj.Name) Called with parentheses (obj.GetName())
Accessors Uses get/set keywords Uses regular parameters and return type
Parameters No explicit parameters (except implicit value in set) Can have any number of parameters
Return get returns property type; set returns void Can return any type or void
Intent "What is this object's state?" "What can this object do?"

Part C: Automatic Property vs Other Types of Properties

C# provides several types of properties. The key difference is whether the compiler generates the backing field automatically or the developer explicitly defines it with custom logic.

1. Automatic Property (Auto-Implemented)

The compiler automatically generates the backing field. No custom logic in accessors. Most concise form. 

class Product
{
    public int Id { get; set; }       // read-write auto property
    public string Name { get; set; }  // read-write auto property
}

2. Full Property (with Backing Field)

Explicit backing field with custom logic in accessors such as validation, computation, or change notification. 

class Product
{
    private double price;

    public double Price
    {
        get { return price; }
        set
        {
            if (value < 0)
                throw new ArgumentException("Price cannot be negative");
            price = value;
        }
    }
}

3. Read-Only Property

Only a get accessor — no set. Used for computed values or immutable state. 

class Rectangle
{
    public double Length { get; set; }
    public double Breadth { get; set; }

    public double Area   // Read-only computed property
    {
        get { return Length * Breadth; }
    }
}

4. Expression-Bodied Property (C# 6+)

Concise syntax using => for read-only or computed properties. 

class Rectangle
{
    public double Length { get; set; }
    public double Breadth { get; set; }
    public double Area => Length * Breadth;  // expression-bodied
}

Complete Example — All Property Types in Action 

class Student
{
    // 1. Automatic property
    public int RollNo { get; set; }

    // 2. Auto property with private set (read-only outside class)
    public string Name { get; private set; }

    // 3. Full property — custom validation
    private int age;
    public int Age
    {
        get { return age; }
        set
        {
            if (value < 1 || value > 150)
                throw new Exception("Invalid age");
            age = value;
        }
    }

    // 4. Read-only computed property
    public string Category
    {
        get
        {
            if (Age < 18) return "Minor";
            if (Age < 60) return "Adult";
            return "Senior";
        }
    }

    // 5. Expression-bodied read-only property
    public string Description =>
        $"Roll: {RollNo}, Name: {Name}, Age: {Age}";

    public Student(int roll, string name, int age)
    {
        RollNo = roll;
        Name = name;
        Age = age;
    }
}

class Test
{
    static void Main()
    {
        Student s = new Student(101, "Alice", 20);
        Console.WriteLine(s.Description);
        Console.WriteLine(s.Category);
    }
}

Output:
Roll: 101, Name: Alice, Age: 20
Adult

Summary

Property Type Backing Field Use Case
Automatic Compiler-generated Simple get/set with no logic
Full Property Explicit (developer-defined) Validation, computed values, change notification
Read-Only Optional Immutable state or computed values
Expression-Bodied Optional Concise read-only computed properties

Properties are function members that encapsulate state with field-like access syntax but method-like internal implementation, whereas methods are function members that perform actions.

5. Define constructor. Explain different types of constructors used in C# with example.

Part A: Definition of Constructor

A constructor in C# is a block of code similar to a method that's called when an instance of an object is created. It is used to initialize the data members of new objects generally.

  • A constructor doesn’t have a return type.
  • The name of the constructor must be the same as the name of the class.
  • Unlike methods, constructors are not considered members of a class.
  • A constructor is called automatically when a new instance of an object is created.

Part B: Different Types of Constructors in C#

According to the .NET Framework, C# provides various types of constructors to initialize objects:

1. Default Constructor

The default constructor is a constructor that is automatically generated in the absence of explicit constructors (i.e. no user-defined constructor). It is sometimes called a nullary constructor. It initializes all member variables to zero or default values. Once you define your own constructor, the default constructor is no longer used. 

class ClassName
{
    public ClassName() 
    {
        // Default constructor body
    }
}

2. Instance Constructors (Parameterized Constructors)

Constructors run initialization code on a class or struct. An instance constructor is defined like a method, except that the method name and return type are reduced to the name of the enclosing type. They can take parameters to initialize fields with specific values. 

public class Panda
{
    private string name;  // Define field

    public Panda(string n) // Define constructor
    {
        name = n;         // Initialization code (set up field)
    }
}

// Call constructor
Panda p = new Panda("Petey");

3. Overloaded Constructors

A class or struct may overload constructors (have multiple constructors with different parameters). To avoid code duplication, one constructor may call another using the this keyword. When one constructor calls another, the called constructor executes first. 

public class Wine
{
    public decimal Price;
    public int Year;

    public Wine(decimal price) 
    { 
        Price = price; 
    }

    public Wine(decimal price, int year) : this(price) 
    { 
        Year = year; 
    }
}

4. Static Constructor

A static constructor is used to perform a particular action only once throughout the application. It is invoked automatically by the CLR before the first instance is created or any static members are referenced. It cannot accept any parameters or access modifiers (it is parameterless). Only one static constructor is allowed per class. 

class User
{
    // Static Constructor
    static User()
    {
        Console.WriteLine("I am Static Constructor");
    }

    // Default Constructor
    public User()
    {
        Console.WriteLine("I am Default Constructor");
    }
}
6. a) What is LINQ? Write a program to select employees whose salary is greater than 20000 and whose address is kathmandu using LINQ
b) Write a C# program to show insert and select operation in database.

Part A: LINQ and Employee Selection

LINQ (Language Integrated Query) is a uniform query syntax in C# to retrieve data from different sources and formats. It is integrated in C#, thereby eliminating the mismatch between programming languages and databases, as well as providing a single querying interface for different types of data sources.

Just as SQL is used to save and retrieve data from a relational database, LINQ is a structured query syntax built into C# to retrieve data from different types of data sources such as collections, ADO.NET DataSets, XML Docs, web services, MS SQL Server, and other databases. LINQ queries return results as objects, enabling an object-oriented approach without worrying about transforming different formats of results into objects.

Program to select employees using LINQ: 

using System;
using System.Collections.Generic;
using System.Linq;

class Employee
{
    public string Name { get; set; }
    public decimal Salary { get; set; }
    public string Address { get; set; }
}

class Program
{
    static void Main()
    {
        // 1. Data source
        List<Employee> employees = new List<Employee>()
        {
            new Employee { Name = "Ram", Salary = 25000, Address = "Kathmandu" },
            new Employee { Name = "Shyam", Salary = 18000, Address = "Pokhara" },
            new Employee { Name = "Hari", Salary = 30000, Address = "Kathmandu" },
            new Employee { Name = "Sita", Salary = 22000, Address = "Lalitpur" }
        };

        // 2. LINQ Query to filter employees
        var selectedEmployees = from emp in employees
                                where emp.Salary > 20000 && emp.Address == "Kathmandu"
                                select emp;

        // 3. Query execution
        Console.WriteLine("Employees with salary > 20000 and address in Kathmandu:");
        foreach (var emp in selectedEmployees)
        {
            Console.WriteLine($"Name: {emp.Name}, Salary: {emp.Salary}");
        }
        
        Console.ReadLine();
    }
}

Below is a C# program demonstrating how to perform an INSERT and a SELECT operation using ADO.NET with SQL Server classes (SqlConnection, SqlCommand, SqlDataAdapter, and DataTable). 

using System;
using System.Data;
using System.Data.SqlClient;

namespace DatabaseTest
{

    {

        }

        void InsertRecord(string name, string address)
        {
            string sql = "INSERT INTO tblStudent (name, address) VALUES ('" + name + "', '" + address + "')";
            command = new SqlCommand(sql, conn);
            command.ExecuteNonQuery();
            Console.WriteLine("Operation Performed Successfully !");
        }

        {
            string sql = "SELECT * FROM tblStudent";
            command = new SqlCommand(sql, conn);

7. What is generics? List different types of generic classes. Explain delegate with example.

Part A: Generics and Types of Generic Classes

Generics in C# and .NET provide many of the benefits of strongly-typed collections, as well as a higher quality and performance boost for code. They are similar to C++ templates, but .NET Generics are not limited to classes—they can also be implemented with Interfaces, Delegates, and Methods. They allow you to define classes, methods, and structures with placeholders (type parameters, e.g., <T>) for the data types they store or use, ensuring type safety without the overhead of boxing and unboxing.

Different Types of Generic Classes

The System.Collections.Generic namespace defines several core generic classes:

Generic Class Description
Collection<T> Provides the base class for a generic collection.
Dictionary<TKey, TValue> Represents a generic collection of key/value pairs.
List<T> Represents a dynamically resizable list of strongly typed objects that can be accessed by index.
Queue<T> Represents a first-in, first-out (FIFO) generic collection of objects.
Stack<T> Represents a variable size last-in-first-out (LIFO) generic collection of instances of the same specified type.

Part B: Delegate with Example

A delegate is an object that knows how to call a method. It is a type that defines the kind of method that delegate instances can call (specifically, the return type and parameter types). All delegates are implicitly derived from the System.Delegate class.

Delegates are primarily used for implementing events and call-back methods. In simple words, a delegate represents a reference to a method with a particular parameter list and return type, allowing the method to be passed as an argument or invoked dynamically.

Example of a Delegate: 

using System;

// 1. Declare the delegate
public delegate int MyDelegate(int x); 

class DelegateTest
{
    // 2. Create a method that matches the delegate signature
    static int MyMethod(int x)
    {
        return x * x;
    }
 
    static void Main(string[] args)
    {
        // 3. Instantiate the delegate
        MyDelegate del = new MyDelegate(MyMethod);
        
        // 4. Invoke the delegate
        int res = del(5); // 25
        
        Console.WriteLine("Result is : " + res);
        Console.ReadKey();
    }
}
8. What do you mean by lambda expression? Explain different types of lambda expression used in C# with example.

Lambda Expression

Lambda expressions in C# are used like anonymous functions, with the difference that in Lambda expressions you don’t need to specify the type of the value that you input thus making it more flexible to use.

The => is the lambda operator which is used in all lambda expressions. The Lambda expression is divided into two parts, the left side is the input and the right is the expression.

Types of Lambda Expressions

The Lambda Expressions can be of two types:

  • 1. Expression Lambda: Consists of the input and the expression.
    Syntax: input => expression;
  • 2. Statement Lambda: Consists of the input and a set of statements to be executed. Unlike an expression lambda, a statement lambda can contain multiple statements separated by semicolons. It is used with delegates.
    Syntax: input => { statements };

Example

Basic example of lambda expression in C#:

using System;

class LambdaTest
{
    static int test1() => 5;
    static int test2(int x) => x + 10;

    static void Main(string[] args)
    {
        int x = test1();
        int res = test2(x);
        Console.WriteLine("Result is: " + res);
    }
}

Output:

Result is: 15

Part B: Exception Handling in ASP.NET

Exception handling in ASP.NET is primarily done using try, catch, and finally blocks to handle runtime errors gracefully without crashing the application.

Here is an example program demonstrating a division operation with exception handling in the code-behind of an ASP.NET page:

using System;

namespace WebApp
{
    public partial class ExceptionHandlingDemo : System.Web.UI.Page
    {
        protected void btnCalculate_Click(object sender, EventArgs e)
        {
            try
            {
                // Assume txtNumerator and txtDenominator are text boxes on the ASP.NET page
                int numerator = Convert.ToInt32(txtNumerator.Text);
                int denominator = Convert.ToInt32(txtDenominator.Text);
                
                int result = numerator / denominator;
                lblResult.Text = "Result: " + result.ToString();
            }
            catch (DivideByZeroException)
            {
                // Specific exception catch block for division by zero
                lblResult.Text = "Error: Cannot divide by zero.";
            }
            catch (FormatException)
            {
                // Specific exception catch block for invalid format (e.g., entering letters instead of numbers)
                lblResult.Text = "Error: Please enter valid numbers.";
            }
            catch (Exception ex)
            {
                // General exception catch block to catch any other errors
                lblResult.Text = "An unexpected error occurred: " + ex.Message;
            }
            finally
            {
                // This block always executes, regardless of whether an exception occurred or not
                // Usually used for cleanup operations
                Console.WriteLine("Operation attempted.");
            }
        }
    }
}
10. a) How virtual method is used to achieve dynamic binding in C#? Explain with the help of suitable program.

Part A: Virtual Methods and Dynamic Binding

A virtual method is a method that can be redefined in derived classes. It has an implementation in a base class as well as the derived class. A virtual method is created in the base class using the virtual keyword and overridden in the derived class using the override keyword.

Dynamic binding (also known as late binding) occurs when a base class reference points to a derived class object. At runtime, the CLR checks the actual object type and calls the overridden method in the derived class instead of the base class method.

Example of Dynamic Binding:

using System;

class Vir
{
    public virtual void message()
    {
        Console.WriteLine("This is test (Base Class)");
    }
}

class Vir1 : Vir
{
    public override void message()
    {
        Console.WriteLine("This is test1 (Derived Class)");
    }
}

class Program
{
    static void Main(string[] args)
    {
        // Base class reference pointing to a derived class object
        Vir obj = new Vir1();
        
        // Dynamic Binding: The derived class method is called at runtime
        obj.message();
        
        Console.ReadKey();
    }
}

Part B: Operator Overloading

Operator overloading gives the ability to use the same operator to do various operations. It provides additional capabilities to C# operators when they are applied to user-defined data types (like classes and structs). It enables developers to make user-defined implementations of operations where one or both of the operands are of a user-defined class.

An overloaded binary operator must take two arguments; at least one of them must be of the type class or struct in which the operation is defined.

Example: Overloading the Binary '+' Operator

using System;

class Complex
{
    private int x;
    private int y;

    public Complex() {}

    public Complex(int i, int j)
    {
        x = i;
        y = j;
    }

    public void ShowXY()
    {
        Console.WriteLine("{0} {1}", x, y);
    }

    // Overloading the binary '+' operator
    public static Complex operator +(Complex c1, Complex c2)
    {
        Complex temp = new Complex();
        temp.x = c1.x + c2.x;
        temp.y = c1.y + c2.y;
        return temp;
    }
}

class MyClient
{
    public static void Main()
    {
        Complex c1 = new Complex(10, 20);
        Complex c2 = new Complex(20, 30);
        Complex c3 = new Complex();
        
        // The overloaded '+' operator is used here
        c3 = c1 + c2;
        
        c3.ShowXY(); // Displays 30 & 50
    }
}
