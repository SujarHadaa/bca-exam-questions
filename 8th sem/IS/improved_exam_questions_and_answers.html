Improved Information Security - Questions and Answers

Unit I: Overview of Computer security

What is Information Security? List and briefly define fundamental security design principles. (2022)

This answer defines Information Security and outlines the core principles that guide the design of secure systems.

Introduction to Information Security

Information Security, often abbreviated as InfoSec, is the practice of protecting information from unauthorized access, use, disclosure, alteration, or destruction. The primary goal of InfoSec is to ensure the Confidentiality, Integrity, and Availability (CIA Triad) of information.

Fundamental Security Design Principles

  • Economy of Mechanism: Security mechanisms should be as simple as possible. Simplicity reduces the chance of errors and makes it easier to verify the design.
  • Fail-Safe Defaults: Access decisions should be based on permission rather than exclusion. This means that access is denied by default, and only granted explicitly.
  • Complete Mediation: Every access to every object must be checked for authority. This prevents attackers from bypassing security checks.
  • Open Design: The design of a security mechanism should be open to public scrutiny. Security should not depend on the secrecy of the design itself.
  • Separation of Privilege: A system should not grant permission based on a single condition. Requiring multiple conditions to be met for access enhances security.
  • Least Privilege: A user or program should have the minimum set of privileges necessary to perform its task. This limits the damage that can be caused by an attacker.
  • Least Common Mechanism: Minimize the mechanisms that are shared among users. Shared mechanisms can be a channel for information leakage.
  • Psychological Acceptability: Security mechanisms should be easy to use. If a mechanism is too cumbersome, users may be tempted to bypass it.

In conclusion, Information Security is a critical aspect of modern computing, and by following these fundamental design principles, we can create more robust and secure systems.

Unit II: Cryptographic Algorithms

What is a cryptosystem? Explain the concepts of Vigenere and Rail Fence Ciphers in detail. (2022)

This answer defines a cryptosystem and explains two classical ciphers: the Vigenere cipher and the Rail Fence cipher.

Cryptosystem

A cryptosystem is a set of cryptographic algorithms used to perform encryption and decryption. It includes the plaintext, ciphertext, encryption algorithm, decryption algorithm, and the key or keys.

Vigenere Cipher

  • Type: Polyalphabetic substitution cipher.
  • Method: It uses a keyword to shift the letters of the plaintext. Each letter of the keyword corresponds to a Caesar cipher shift.
  • Example: If the plaintext is "ATTACKATDAWN" and the keyword is "LEMON", the first 'A' is encrypted using the 'L' shift, the first 'T' with the 'E' shift, and so on.

Rail Fence Cipher

  • Type: Transposition cipher.
  • Method: The plaintext is written diagonally on a number of "rails". The ciphertext is then read off row by row.
  • Example: For the plaintext "WEAREDISCOVEREDFLEEATONCE" with 3 rails, the ciphertext would be "WECRLTEERDSOEEFEAOCAIVDEN".

In summary, a cryptosystem provides the framework for encryption, and classical ciphers like Vigenere and Rail Fence, while simple, form the basis of modern cryptography.

Unit III: Message Authentication and Hash Functions

Explain HMAC with its objectives. (2022)

This answer explains HMAC, a mechanism for message authentication using cryptographic hash functions, and its objectives.

HMAC (Hash-based Message Authentication Code)

HMAC is a type of Message Authentication Code (MAC) that is calculated using a cryptographic hash function in combination with a secret key. It provides both data integrity and authenticity of a message.

Objectives of HMAC

  • To use hash functions without modifications: HMAC is designed to be a wrapper around a hash function, so it can be used with any standard cryptographic hash function.
  • To allow for easy replaceability of the hash function: If a hash function is found to be insecure, it can be easily replaced with a more secure one.
  • To preserve the original performance of the hash function: HMAC does not significantly slow down the hashing process.
  • To use and handle keys in a simple way: The key management for HMAC is straightforward.

In conclusion, HMAC is a widely used and secure mechanism for message authentication that is flexible, efficient, and easy to implement.

Unit IV: User Authentication

What are the vulnerabilities of passwords? Explain different strategies used for the selection of effective password. (2022)

This answer discusses the vulnerabilities of password-based authentication and provides strategies for selecting effective passwords.

Password Vulnerabilities

  • Brute-Force Attack: Trying all possible combinations of characters.
  • Dictionary Attack: Trying words from a dictionary.
  • Social Engineering: Tricking users into revealing their passwords.
  • Shoulder Surfing: Observing someone typing their password.
  • Keylogging: Recording keystrokes on a compromised computer.

Strategies for Effective Password Selection

  • Length: Use a long password (at least 12 characters).
  • Complexity: Use a mix of uppercase and lowercase letters, numbers, and symbols.
  • Uniqueness: Use a different password for each account.
  • Avoid personal information: Do not use your name, birthdate, or other personal information.
  • Use a password manager: To generate and store strong, unique passwords.

In conclusion, while passwords are a common form of authentication, they have many vulnerabilities. By following these strategies, users can create more secure passwords and reduce the risk of unauthorized access.

Unit V: Access Control

Define access right. Explain the concept of trust framework. (2022)

This answer defines an access right and explains the concept of a trust framework in the context of access control.

Access Right

An access right is a permission granted to a user or a process to perform a specific action on a resource. Examples of access rights include the permission to read, write, or execute a file.

Trust Framework

A trust framework is a set of policies, standards, and agreements that allow different organizations to rely on each other's digital identities. It provides a legal and technical framework for establishing and maintaining trust between different entities in a distributed system.

In summary, access rights are the basic building blocks of access control, while trust frameworks provide a way to manage access control in a distributed and federated environment.

Unit VI: Malicious Software and Intrusion

Describe about Honeypot with the types of honeypots that may be deployed. (2022)

This answer describes a honeypot, a decoy system used to attract and study attackers, and its different types.

Honeypot

A honeypot is a security mechanism that is designed to be a target for attackers. It is a decoy system that is set up to gather information about attackers and their methods. By studying the behavior of attackers in a controlled environment, security professionals can learn how to better protect their real systems.

Types of Honeypots

  • Low-interaction honeypots: These honeypots simulate a limited number of services and are easy to set up. They are used to detect and log basic attacks.
  • High-interaction honeypots: These honeypots provide a real, but controlled, environment for attackers. They are more complex to set up but provide more detailed information about attackers.
  • Research honeypots: These honeypots are used to study new attack techniques and malware.
  • Production honeypots: These honeypots are used to protect production systems by diverting attackers away from real targets.

In conclusion, honeypots are a valuable tool for understanding and defending against cyberattacks. By using different types of honeypots, organizations can gain insights into the threat landscape and improve their security posture.

Unit VII: Network Security

Explain the terms SSL, TSL and handshake protocol. (2022)

This answer explains SSL, TLS, and the handshake protocol, which are fundamental to securing network communications.

SSL (Secure Sockets Layer)

SSL is a cryptographic protocol that provides security for communications over a computer network. It was the predecessor to TLS and is now considered insecure.

TLS (Transport Layer Security)

TLS is the successor to SSL. It provides improved security and is widely used in applications such as web browsing, email, and instant messaging.

Handshake Protocol

The handshake protocol is a mechanism used in TLS to negotiate the security parameters of a connection. The client and server use the handshake protocol to agree on the version of the protocol to use, the cryptographic algorithms to use, and to authenticate each other.

In summary, SSL and TLS are protocols that provide security for network communications, and the handshake protocol is a key part of TLS that allows the client and server to establish a secure connection.

Unit VIII: Security Auditing

What is a security audit? What is its importance? Explain the security auditing architecture in detail. (2022)

This answer defines a security audit, explains its importance, and describes the architecture of a security auditing system.

Security Audit

A security audit is a systematic and measurable technical assessment of a system or application. It is a manual or automated process that is used to verify compliance with a set of security policies and standards.

Importance of Security Audits

  • Identify vulnerabilities: Security audits can help to identify vulnerabilities in a system before they can be exploited by attackers.
  • Improve security posture: The findings of a security audit can be used to improve the security of a system.
  • Compliance: Security audits can be used to verify compliance with industry and government regulations.

Security Auditing Architecture

A security auditing architecture consists of the following components:

  • Auditing policy: A set of rules that defines what is to be audited.
  • Auditing mechanism: The tools and techniques used to collect and analyze audit data.
  • Auditing trail: A record of the audited events.

In conclusion, security audits are an essential part of a comprehensive security program. By regularly auditing their systems, organizations can identify and mitigate security risks, and improve their overall security posture.
