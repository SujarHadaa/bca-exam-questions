Improved Information Security - Notes

Unit I: Overview of Computer security

Computer Security Concepts

CIA Triad: Confidentiality, Integrity, Availability.

Threats: Potential for security violation.

Attacks: Deliberate assault on system security.

Assets: Information, hardware, software.

Security Design Principles

Least Privilege: Minimum necessary access.

Fail-Safe Defaults: Deny by default.

Open Design: No security through obscurity.

Unit II: Cryptographic Algorithms

Classical Ciphers

Vigenere: Polyalphabetic substitution (uses keyword).

Rail Fence: Transposition cipher (writes diagonally).

Modern Ciphers

Symmetric: Same key for encryption and decryption (e.g., DES, AES).

Asymmetric: Different keys for encryption and decryption (e.g., RSA, Diffie-Hellman).

Unit III: Message Authentication and Hash Functions

Message Authentication

MAC (Message Authentication Code): A short piece of information used to authenticate a message.

HMAC: A type of MAC that uses a hash function and a secret key.

Hash Functions

MD5, SHA-1: Older hash functions (less secure).

Digital Signatures: Provide authentication, integrity, and non-repudiation.

Unit IV: User Authentication

Authentication Methods

Password-based: Something you know.

Token-based: Something you have.

Biometric: Something you are.

Two-Factor Authentication (2FA): Combines two different methods.

Unit V: Access Control

Access Control Models

Discretionary Access Control (DAC): The owner of a resource determines who has access.

Role-Based Access Control (RBAC): Access is based on the user's role in the organization.

Attribute-Based Access Control (ABAC): Access is based on attributes of the user, resource, and environment.

Unit VI: Malicious Software and Intrusion

Malicious Software (Malware)

Virus: Attaches itself to a program and spreads to other programs.

Worm: A self-replicating program that spreads across a network.

Trojan Horse: A program that appears to be harmless but has a malicious purpose.

Intrusion Detection System (IDS)

Signature-based IDS: Detects known attacks by looking for specific patterns.

Anomaly-based IDS: Detects unknown attacks by looking for deviations from normal behavior.

Unit VII: Network Security

Network Security Protocols

SSL/TLS: Secure communication over a network.

IPSec: Secure communication at the IP layer.

Firewall: A network security device that monitors and filters incoming and outgoing network traffic.

Unit VIII: Security Auditing

Security Auditing

Security Audit: A systematic evaluation of the security of a system.

Audit Trail: A record of system activity.

Logging: The process of collecting and storing audit data.
