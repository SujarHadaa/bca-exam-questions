2. What is security attack? Explain different type of active attack.

Security Attack: Any action that compromises the security of information owned by an organization or an individual.

Active Attacks

These attacks involve the modification of a data stream or the creation of a false stream. They try to alter system resources or affect their operations. They can be subdivided into four categories:

  • Masquerade: Happens when an unauthorized entity pretends to be a legitimate entity to gain access.
  • Replay: Involves capturing a legitimate data transmission and resending it later to produce an unauthorized effect.
  • Modification of messages: Happens when a legitimate message is altered, delayed, or reordered to cause an unauthorized effect.
  • Denial of service (DoS): Prevents or inhibits the normal use or management of communication facilities or system resources.
3. Find the value of 7^2019 mod 11 by using Fermat's little theorem. Define Euler totient function with example.

Part 1: Using Fermat's Little Theorem

Fermat's Little Theorem states that if p is a prime number, then for any integer a not divisible by p, the following equation holds true:

a(p-1) ≡ 1 (mod p)

Here is the step-by-step numerical solution to find 72019 mod 11:

// 1. Identify given values
a = 7
p = 11  // 11 is a prime number

// 2. Apply Fermat's Little Theorem: a^(p-1) ≡ 1 (mod p)
7^(11-1) ≡ 1 (mod 11)
7^10 ≡ 1 (mod 11)

// 3. Break down the large power (2019) into multiples of 10
2019 = 10 * 201 + 9

// 4. Rewrite the expression
7^2019 = 7^(10 * 201 + 9)
       = (7^10)^201 * 7^9

// 5. Substitute 7^10 with 1
       ≡ (1)^201 * 7^9 (mod 11)
       ≡ 7^9 (mod 11)

// 6. Calculate 7^9 mod 11 using smaller powers
7^1 ≡ 7 (mod 11)
7^2 = 49 ≡ 5 (mod 11)            // because 49 = 11*4 + 5
7^4 = (7^2)^2 ≡ 5^2 = 25 ≡ 3 (mod 11)
7^8 = (7^4)^2 ≡ 3^2 = 9 (mod 11)

// 7. Combine results to get 7^9
7^9 = 7^8 * 7^1 
    ≡ 9 * 7 = 63
   
63 = 11 * 5 + 8 ≡ 8 (mod 11)

// Final Answer
Result = 8

Part 2: Euler's Totient Function

Euler's Totient Function, written as Φ(n) (phi of n), is a mathematical function that counts how many positive integers up to n are relatively prime (or coprime) to n.

Two numbers are relatively prime if their Greatest Common Divisor (GCD) is 1, meaning they share no common factors other than 1.

Here is a simple example to understand this concept:

// Example: Let's find Φ(6)
n = 6

// Step 1: List all positive integers up to 6
// They are: 1, 2, 3, 4, 5, 6

// Step 2: Find the GCD of each number with 6
GCD(1, 6) = 1  -> (Relatively Prime)
GCD(2, 6) = 2
GCD(3, 6) = 3
GCD(4, 6) = 2
GCD(5, 6) = 1  -> (Relatively Prime)
GCD(6, 6) = 6

// Step 3: Count the numbers that have a GCD of 1
// The numbers are 1 and 5. There are exactly 2 such numbers.

// Final Answer for the example
Φ(6) = 2
4. What is Galois field? Divide 5x² + 4x + 6 by 2x + 1 over GF (7).

Part 1: What is a Galois Field?

A Galois Field (also known as a Finite Field) is a set that contains a finite number of elements in which you can add, subtract, multiply, and divide (except by zero) without ever leaving the set.

It is denoted as GF(p) or GF(pn), where p is a prime number. In a simple Galois field like GF(7), the elements are just the numbers from 0 to 6, and all mathematical operations wrap around using modulo 7. It is heavily used in computer security and cryptography because it keeps numbers within a fixed size.

Part 2: Polynomial Division over GF(7)

Here is the step-by-step division of the polynomials using Modulo 7 arithmetic:

// Given Polynomials
Dividend = 5x² + 4x + 6
Divisor = 2x + 1

// In GF(7), all numbers are modulo 7. 
// First, we need to know how to divide by 2 in GF(7). 
// Dividing by 2 is the same as multiplying by the inverse of 2.
// 2 * 4 = 8 ≡ 1 (mod 7). So, the inverse of 2 is 4.

// --- STEP 1: Divide the leading terms ---
(5x²) / (2x) = (5 * 4)x = 20x ≡ 6x (mod 7)
// First term of the Quotient is 6x.

// --- STEP 2: Multiply and Subtract ---
Multiply: (2x + 1) * 6x = 12x² + 6x ≡ 5x² + 6x (mod 7)
Subtract: (5x² + 4x + 6) - (5x² + 6x) 
          = (4x - 6x) + 6 
          = -2x + 6
          
// Since -2 ≡ 5 (mod 7), the remainder becomes:
New Dividend = 5x + 6

// --- STEP 3: Divide the new leading terms ---
(5x) / (2x) = (5 * 4) = 20 ≡ 6 (mod 7)
// Second term of the Quotient is 6.

// --- STEP 4: Multiply and Subtract ---
Multiply: (2x + 1) * 6 = 12x + 6 ≡ 5x + 6 (mod 7)
Subtract: (5x + 6) - (5x + 6) = 0

// Final Answer
Quotient = 6x + 6
Remainder = 0
5. What is digital signature? Describe approaches of DSS.

Part 1: What is a Digital Signature?

A Digital Signature is a mathematical technique used to validate the authenticity and integrity of a digital document or message. It acts as the digital equivalent of a handwritten signature or a stamped seal but offers far more security.

A digital signature provides three primary guarantees:

  • Authenticity: It proves that the message was indeed created and sent by the claimed sender.
  • Integrity: It ensures that the message was not altered or tampered with during transmission.
  • Non-repudiation: The sender cannot deny having sent the message later on.

Part 2: Approaches of DSS (Digital Signature Standard)

The Digital Signature Standard (DSS) uses a public-key cryptography approach. The entire process is divided into two main phases: the Signing Phase and the Verification Phase.

  • 1. The Signing Phase (Done by the Sender):
    - First, a Hash Function is applied to the original message to generate a fixed-size Message Digest (or hash value).
    - Then, this message digest is encrypted using the sender's Private Key.
    - This encrypted digest is the Digital Signature.
    - Finally, the digital signature is attached to the original message and sent over the network.
  • 2. The Verification Phase (Done by the Receiver):
    - The receiver gets both the original message and the digital signature.
    - The receiver applies the same Hash Function to the received message to generate a New Message Digest.
    - Concurrently, the receiver decrypts the digital signature using the sender's Public Key to retrieve the Original Message Digest.
    - If the New Message Digest perfectly matches the Original Message Digest, the signature is considered valid.
6. What is firewall? Explain different type of firewall.

Part 1: What is a Firewall?

A Firewall is a network security system (either hardware or software) that monitors and controls incoming and outgoing network traffic based on predetermined security rules.

It acts as a protective barrier or a "gatekeeper" between a trusted, secure internal network (like a company's private network) and an untrusted external network (like the public Internet), preventing unauthorized access and malicious attacks.

Part 2: Different Types of Firewalls

Firewalls can be categorized based on how they inspect and filter the network traffic:

  • 1. Packet Filtering Firewall:
    - The most basic and oldest type of firewall.
    - It examines individual packets of data in isolation and filters them based on basic information like source/destination IP address, port numbers, and protocols.
    - It is fast but does not check the actual content (payload) of the packet, making it less secure against advanced attacks.
  • 2. Stateful Inspection Firewall:
    - Also known as dynamic packet filtering.
    - It keeps track of the "state" of active network connections (e.g., a TCP connection).
    - It only allows packets that are part of an already established and verified connection, making it much more secure than basic packet filtering.
  • 3. Application-Level Gateway (Proxy Firewall):
    - Operates at the Application layer and acts as an intermediary (proxy) between the user and the internet.
    - Instead of a direct connection, the client connects to the proxy, and the proxy connects to the destination.
    - It provides deep inspection of the actual data payload (like HTTP or FTP traffic) but can slow down network performance.
  • 4. Circuit-Level Gateway:
    - Works at the Session layer of the network model.
    - It simply verifies the TCP handshake between packets to ensure the session is legitimate before a connection is established.
    - Once the connection is approved, it doesn't inspect the data further.
  • 5. Next-Generation Firewall (NGFW):
    - The most advanced modern firewall.
    - It combines traditional firewall features (like stateful inspection) with deep packet inspection (DPI), intrusion prevention systems (IPS), and advanced malware detection to block modern sophisticated threats.
7. What are user authentication principles? Explain about Role based and Attribute based access control.

Part 1: User Authentication Principles

User Authentication is the fundamental process of verifying the identity claimed by a user or system entity. It is the first line of defense in computer security.

The principles of authentication (how a user can prove who they are) are broadly categorized into four primary factors:

  • Something you know: Requires a secret piece of information known only to the user. (e.g., Passwords, PINs, or security questions).
  • Something you have: Requires physical possession of a token or device. (e.g., Smart cards, RSA physical tokens, or an authenticator app on a smartphone).
  • Something you are (Static Biometrics): Uses physical characteristics unique to the individual. (e.g., Fingerprints, retina scans, or facial recognition).
  • Something you do (Dynamic Biometrics): Uses behavioral characteristics. (e.g., Voice patterns, typing rhythm, or a handwritten signature).

Combining two or more of these principles creates Multi-Factor Authentication (MFA), which significantly increases security.

Part 2: Role-Based Access Control (RBAC) vs. Attribute-Based Access Control (ABAC)

Once a user is authenticated, Access Control dictates what resources they are allowed to use. RBAC and ABAC are two prominent access control models:

Feature Role-Based Access Control (RBAC) Attribute-Based Access Control (ABAC)
Core Concept Permissions are tied to Roles (e.g., "Manager", "Clerk"). Users are assigned to these roles. Permissions are granted based on Attributes (characteristics of the user, resource, or environment).
Flexibility & Granularity Less granular. It is rigid because it only looks at the user's job title or group. Highly granular and flexible. It can evaluate complex rules (e.g., time of day, location, clearance level).
Administration Simple to manage in standard organizations. When a user changes jobs, you just change their role. More complex to set up and manage due to the need to define complex logical policies and many attributes.
Example "All Doctors can access patient records." "A Doctor can access patient records ONLY during hospital hours AND if they are on the hospital's network."
8. What is security audit trial? What are requirements for security auditing? Explain.

Part 1: Security Audit Trail

A Security Audit Trail (or audit log) is a chronological record of system activities that is used to reconstruct and examine the sequence of events that have occurred.

It acts as a digital footprint, providing a reliable means to trace the activities of users, track changes to data, and to detect, investigate, and recover from security breaches or system failures.
Layman analogy: A detailed diary or a CCTV camera log that records exactly who entered a room, what time they entered, and what they did while inside.

Part 2: Requirements for Security Auditing

For a security audit trail to be effective and trustworthy, it must meet several critical requirements:

  • Completeness: The audit trail should capture all relevant and significant security events without missing anything important.
  • Accuracy: The information recorded must be precise, correct, and reliable, showing exactly what happened without errors.
  • Integrity: The audit log must be tamper-proof. It should be strictly protected from unauthorized modification or deletion so attackers cannot cover their tracks.
  • Confidentiality: The audit trail often contains sensitive information. It should be protected from unauthorized access or disclosure to ensure only authorized auditors can read it.
  • Availability: The audit trail should always be readily available and accessible to authorized personnel whenever an investigation or review is needed.

Reverse the rules: Shift left for rows, up for columns, and same opposite corners for rectangles.

10. How does the Kerberos authentication works? Explain the concept of trust frameworks.

Part 1: Kerberos Authentication

Kerberos is a computer network authentication protocol that works on the basis of 'tickets'. It is designed to provide strong authentication for client/server applications by allowing nodes communicating over a non-secure network to securely prove their identity to one another.

Here is how the Kerberos process generally works:

  • Trusted Third Party: It uses a centralized Authentication Server (AS) and a Ticket Granting Server (TGS), collectively known as the Key Distribution Center (KDC).
  • Symmetric-Key Cryptography: The protocol primarily relies on symmetric encryption, meaning the KDC shares a unique secret key with every user and service on the network.
  • The Process:
    1. The client requests access from the Authentication Server (AS).
    2. The AS verifies the client's identity and issues a Ticket-Granting Ticket (TGT).
    3. The client presents the TGT to the Ticket Granting Server (TGS) to request access to a specific service.
    4. The TGS validates the TGT and issues a Service Ticket.
    5. The client presents the Service Ticket to the actual service/server to gain access.

Layman analogy: Think of it like going to an amusement park. You show your ID at the main gate to get a daily wristband (TGT). Then, instead of showing your ID at every single ride, you just show your wristband to the ride operator (Service Ticket) to get on the ride.

Part 2: Trust Frameworks

A Trust Framework is a formalized set of agreed-upon rules, security policies, standards, and legal agreements that allow different organizations (or security domains) to trust each other's identity and security information.

It essentially creates a common language and standard of security. Key concepts include:

  • Federated Identity Management: Trust frameworks are heavily used in federated systems. This allows a user from Organization A to securely access resources in Organization B using their original login credentials from Organization A.
  • Scalability: Instead of creating one-on-one agreements between every single entity (which is unmanageable), a trust framework allows an entity to join the framework once and automatically trust everyone else within it.
  • Consistency: It ensures that all participants meet a minimum baseline of security and identity verification, reducing the overall risk of unauthorized access across interconnected networks.
11. What is IPSec? What are benefits of IPSec? Explain the IPSec architecture in details.

Part 1: What is IPSec?

IPSec (Internet Protocol Security) is a suite of protocols that provides security for Internet Protocol (IP) communications by authenticating and encrypting each IP packet of a communication session. It operates at the Network Layer (Layer 3) of the OSI model, which means it protects any application traffic running over IP.

Part 2: Benefits of IPSec

  • Confidentiality: Uses strong encryption to ensure that intercepted packets cannot be read by unauthorized parties.
  • Integrity: Uses cryptographic hashing to ensure that the data has not been altered or tampered with in transit.
  • Data Origin Authentication: Verifies that the packets actually came from the claimed sender.
  • Anti-Replay Protection: Uses sequence numbers to detect and reject old, duplicated packets, preventing replay attacks.
  • Transparency: Because it operates at the network layer, it provides security transparently without requiring changes to individual applications.
