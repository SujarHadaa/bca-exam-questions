Unit 3: Designing the User Interface

Q1: What is an event? How many ways to handle events in android programming?

An event in Android is an action that occurs as a result of the user's interaction with the user interface elements. Examples of events include clicking a button, long-pressing an item, or entering text into a field.

There are three primary ways to handle events in Android programming:

  1. Using the android:onClick attribute in the XML layout file: This is the simplest method, suitable for simple click events.
  2. Using an Anonymous Inner Class: This is a more common and flexible approach, allowing you to handle events for any type of listener.
  3. Implementing the Listener Interface in the Activity: This method is useful when you have multiple UI elements that share the same event handling logic.

Code Examples:

1. Using android:onClick in XML

activity_main.xml:

<Button
    android:id="@+id/myButton"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Click Me"
    android:onClick="onButtonClick" />

MainActivity.java:

public class MainActivity extends AppCompatActivity {
    // ...
    public void onButtonClick(View view) {
        Toast.makeText(this, "Button Clicked!", Toast.LENGTH_SHORT).show();
    }
}

2. Using an Anonymous Inner Class

MainActivity.java:

Button myButton = findViewById(R.id.myButton);
myButton.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        Toast.makeText(MainActivity.this, "Button Clicked!", Toast.LENGTH_SHORT).show();
    }
});

3. Implementing the Listener Interface

MainActivity.java:

public class MainActivity extends AppCompatActivity implements View.OnClickListener {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        Button myButton = findViewById(R.id.myButton);
        myButton.setOnClickListener(this);
    }

    @Override
    public void onClick(View v) {
        if (v.getId() == R.id.myButton) {
            Toast.makeText(this, "Button Clicked!", Toast.LENGTH_SHORT).show();
        }
    }
}

Cheat Sheet: Event Handling

  • Event: User interaction with a UI element.
  • Handling Methods:
    1. XML onClick: Simple, for clicks only.
    2. Anonymous Inner Class: Flexible, for any listener.
    3. Implement Interface: Good for multiple elements with the same logic.

Q2: What is the difference between android:layout_gravity and android:gravity attribute in linear layout. Create an android application that will convert distance entered into Kilometers into Meters using Table Layout.

The android:gravity and android:layout_gravity attributes are used to align views, but they have different scopes:

  • android:gravity: This attribute is used to align the content of a view within the view itself. For example, it can align the text within a TextView to the center or right.
  • android:layout_gravity: This attribute is used to align a view (or widget) within its parent layout. For example, it can align a Button to the center or right of a LinearLayout.

Kilometer to Meter Converter App (TableLayout)

activity_main.xml:

<?xml version="1.0" encoding="utf-8"?>
<TableLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:padding="16dp"
    android:stretchColumns="1">

    <TableRow>
        <TextView
            android:text="Kilometers:"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:layout_column="0" />
        <EditText
            android:id="@+id/kmEditText"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:inputType="numberDecimal"
            android:layout_column="1" />
    </TableRow>

    <TableRow>
        <Button
            android:id="@+id/convertButton"
            android:text="Convert"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:layout_column="1" />
    </TableRow>

    <TableRow>
        <TextView
            android:text="Meters:"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:layout_column="0" />
        <TextView
            android:id="@+id/metersTextView"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:layout_column="1"
            android:textSize="18sp" />
    </TableRow>
</TableLayout>

MainActivity.java:

public class MainActivity extends AppCompatActivity {

    private EditText kmEditText;
    private Button convertButton;
    private TextView metersTextView;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        kmEditText = findViewById(R.id.kmEditText);
        convertButton = findViewById(R.id.convertButton);
        metersTextView = findViewById(R.id.metersTextView);

        convertButton.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                String kmString = kmEditText.getText().toString();
                if (!kmString.isEmpty()) {
                    double km = Double.parseDouble(kmString);
                    double meters = km * 1000;
                    metersTextView.setText(String.valueOf(meters));
                }
            }
        });
    }
}

Cheat Sheet: gravity vs. layout_gravity & Converter App

  • gravity: Aligns content inside a view.
  • layout_gravity: Aligns a view within its parent.
  • Converter App:
    • Uses TableLayout for a structured UI.
    • An EditText takes kilometer input.
    • A Button triggers the conversion.
    • A TextView displays the result in meters.

Q3: What are important attributes used in Absolute Layout? Develop a simple calculator application with two input fields for inputting numbers and two buttons for performing multiplication and division and display the result in TextView using Absolute Layout.

AbsoluteLayout is a layout manager that lets you specify the exact x and y coordinates of its children. It is generally discouraged in modern Android development because it does not adapt well to different screen sizes and resolutions.

The most important attributes for AbsoluteLayout are:

  • android:layout_x: Specifies the x-coordinate of the view.
  • android:layout_y: Specifies the y-coordinate of the view.

Simple Calculator App (AbsoluteLayout)

activity_main.xml:

<?xml version="1.0" encoding="utf-8"?>
<AbsoluteLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:padding="16dp">

    <EditText
        android:id="@+id/number1EditText"
        android:layout_width="150dp"
        android:layout_height="wrap_content"
        android:inputType="numberDecimal"
        android:layout_x="20dp"
        android:layout_y="20dp" />

    <EditText
        android:id="@+id/number2EditText"
        android:layout_width="150dp"
        android:layout_height="wrap_content"
        android:inputType="numberDecimal"
        android:layout_x="20dp"
        android:layout_y="80dp" />

    <Button
        android:id="@+id/multiplyButton"
        android:text="*"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_x="20dp"
        android:layout_y="140dp" />

    <Button
        android:id="@+id/divideButton"
        android:text="/"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_x="120dp"
        android:layout_y="140dp" />

    <TextView
        android:id="@+id/resultTextView"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:textSize="24sp"
        android:layout_x="20dp"
        android:layout_y="200dp" />

</AbsoluteLayout>

MainActivity.java:

public class MainActivity extends AppCompatActivity {

    private EditText number1EditText, number2EditText;
    private Button multiplyButton, divideButton;
    private TextView resultTextView;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        number1EditText = findViewById(R.id.number1EditText);
        number2EditText = findViewById(R.id.number2EditText);
        multiplyButton = findViewById(R.id.multiplyButton);
        divideButton = findViewById(R.id.divideButton);
        resultTextView = findViewById(R.id.resultTextView);

        multiplyButton.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                calculate('*');
            }
        });

        divideButton.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                calculate('/');
            }
        });
    }

    private void calculate(char operator) {
        String num1Str = number1EditText.getText().toString();
        String num2Str = number2EditText.getText().toString();

        if (!num1Str.isEmpty() && !num2Str.isEmpty()) {
            double num1 = Double.parseDouble(num1Str);
            double num2 = Double.parseDouble(num2Str);
            double result = 0;

            if (operator == '*') {
                result = num1 * num2;
            } else if (operator == '/') {
                if (num2 != 0) {
                    result = num1 / num2;
                } else {
                    resultTextView.setText("Cannot divide by zero");
                    return;
                }
            }
            resultTextView.setText("Result: " + result);
        }
    }
}

Cheat Sheet: AbsoluteLayout & Calculator App

  • AbsoluteLayout: Positions children using exact x/y coordinates. (Not recommended for general use).
  • Attributes: android:layout_x, android:layout_y.
  • Calculator App:
    • Uses AbsoluteLayout to position two EditTexts, two Buttons, and a TextView.
    • Performs multiplication and division.

Q4: How to create AVD's? How to use spinner in Android? Explain with a suitable example.

Creating an AVD (Android Virtual Device)

An AVD is an emulator configuration that lets you model an actual Android device. You can create and manage AVDs using the AVD Manager in Android Studio.

  1. Open Android Studio.
  2. Go to Tools > AVD Manager.
  3. Click on + Create Virtual Device....
  4. Select a hardware profile (e.g., Pixel 6) and click Next.
  5. Choose a system image (e.g., Tiramisu) and click Next. You may need to download the image first.
  6. Give your AVD a name and configure other settings as needed.
  7. Click Finish.

Using a Spinner in Android

A Spinner is a dropdown list that allows users to select one value from a set.

activity_main.xml:

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical"
    android:padding="16dp">

    <Spinner
        android:id="@+id/planets_spinner"
        android:layout_width="match_parent"
        android:layout_height="wrap_content" />

</LinearLayout>

strings.xml:

<resources>
    <string-array name="planets_array">
        <item>Mercury</item>
        <item>Venus</item>
        <item>Earth</item>
        <item>Mars</item>
        <item>Jupiter</item>
        <item>Saturn</item>
        <item>Uranus</item>
        <item>Neptune</item>
    </string-array>
</resources>

MainActivity.java:

public class MainActivity extends AppCompatActivity implements AdapterView.OnItemSelectedListener {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        Spinner spinner = findViewById(R.id.planets_spinner);
        ArrayAdapter<CharSequence> adapter = ArrayAdapter.createFromResource(this,
                R.array.planets_array, android.R.layout.simple_spinner_item);
        adapter.setDropDownViewResource(android.R.layout.simple_spinner_dropdown_item);
        spinner.setAdapter(adapter);
        spinner.setOnItemSelectedListener(this);
    }

    public void onItemSelected(AdapterView<?> parent, View view,
                               int pos, long id) {
        String selectedPlanet = parent.getItemAtPosition(pos).toString();
        Toast.makeText(this, "Selected: " + selectedPlanet, Toast.LENGTH_SHORT).show();
    }

    public void onNothingSelected(AdapterView<?> parent) {
        // Another interface callback
    }
}

Cheat Sheet: AVDs & Spinners

  • AVD Creation: Tools > AVD Manager > Create Virtual Device.
  • Spinner: A dropdown list for selecting an item.
    • Use an ArrayAdapter to populate the spinner with data (often from a string array in strings.xml).
    • Implement AdapterView.OnItemSelectedListener to handle selections.

Q5: What is Toast? Develop an android application to create an alert dialog to input any number and then calculate the factorial of the number and display the result in Toast.

A Toast is a short, transient notification that appears at the bottom of the screen. It provides simple feedback about an operation in a small popup. It automatically fades out after a certain amount of time.

Factorial Calculator with AlertDialog and Toast

activity_main.xml:

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:gravity="center"
    android:orientation="vertical">

    <Button
        android:id="@+id/showDialogButton"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Calculate Factorial" />

</LinearLayout>

MainActivity.java:

import androidx.appcompat.app.AlertDialog;
import androidx.appcompat.app.AppCompatActivity;
import android.content.DialogInterface;
import android.os.Bundle;
import android.text.InputType;
import android.view.View;
import android.widget.Button;
import android.widget.EditText;
import android.widget.Toast;

public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        Button showDialogButton = findViewById(R.id.showDialogButton);
        showDialogButton.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                showFactorialDialog();
            }
        });
    }

    private void showFactorialDialog() {
        AlertDialog.Builder builder = new AlertDialog.Builder(this);
        builder.setTitle("Factorial Calculator");

        final EditText input = new EditText(this);
        input.setInputType(InputType.TYPE_CLASS_NUMBER);
        builder.setView(input);

        builder.setPositiveButton("Calculate", new DialogInterface.OnClickListener() {
            @Override
            public void onClick(DialogInterface dialog, int which) {
                String numStr = input.getText().toString();
                if (!numStr.isEmpty()) {
                    int num = Integer.parseInt(numStr);
                    long factorial = calculateFactorial(num);
                    Toast.makeText(MainActivity.this, "Factorial of " + num + " is " + factorial, Toast.LENGTH_LONG).show();
                }
            }
        });
        builder.setNegativeButton("Cancel", new DialogInterface.OnClickListener() {
            @Override
            public void onClick(DialogInterface dialog, int which) {
                dialog.cancel();
            }
        });

        builder.show();
    }

    private long calculateFactorial(int n) {
        if (n < 0) return -1; // Or throw an exception
        if (n == 0) return 1;
        long fact = 1;
        for (int i = 1; i <= n; i++) {
            fact *= i;
        }
        return fact;
    }
}

Cheat Sheet: Toast & Factorial App

  • Toast: A simple, short-lived notification. Use Toast.makeText(...).show().
  • Factorial App:
    • A Button shows an AlertDialog.
    • The AlertDialog contains an EditText for number input.
    • The factorial is calculated and displayed in a Toast message.
