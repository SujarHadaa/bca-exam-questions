What is an event? How many ways to handle events in android programming?

An event in Android is an action triggered by the user interacting with the device (e.g., clicking a button, touching the screen, or typing on the keyboard).

Ways to Handle Events:

  1. Using the onClick attribute in XML: Simplest way, where you specify a method name in the XML file and define it in the Activity.
  2. Using an Anonymous Inner Class: Setting an OnClickListener directly on the view in Java/Kotlin code.
  3. Implementing the Listener Interface: Having the Activity class implement the listener (e.g., implements View.OnClickListener).
  4. Using an Inner Class: Creating a separate named inner class that implements the listener interface.
Group B
What are important attributes used in Absolute Layout? Develop a simple calculator application... using Absolute Layout.

Note: AbsoluteLayout is deprecated and discouraged because it doesn't scale well to different screen sizes. However, for exam purposes:

Important Attributes:

  • android:layout_x: Specifies the absolute x-coordinate of the view.
  • android:layout_y: Specifies the absolute y-coordinate of the view.

Calculator Code (Absolute Layout):

<!-- activity_main.xml -->
<AbsoluteLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <EditText android:id="@+id/num1" android:layout_width="150dp" android:layout_height="wrap_content"
        android:layout_x="10dp" android:layout_y="20dp" android:hint="Number 1" />

    <EditText android:id="@+id/num2" android:layout_width="150dp" android:layout_height="wrap_content"
        android:layout_x="10dp" android:layout_y="80dp" android:hint="Number 2" />

    <Button android:id="@+id/btnMul" android:layout_width="wrap_content" android:layout_height="wrap_content"
        android:layout_x="10dp" android:layout_y="140dp" android:text="Multiply" />

    <Button android:id="@+id/btnDiv" android:layout_width="wrap_content" android:layout_height="wrap_content"
        android:layout_x="120dp" android:layout_y="140dp" android:text="Divide" />

    <TextView android:id="@+id/result" android:layout_width="wrap_content" android:layout_height="wrap_content"
        android:layout_x="10dp" android:layout_y="200dp" android:text="Result: " android:textSize="20sp" />
</AbsoluteLayout>

// MainActivity.java Snippet
btnMul.setOnClickListener(v -> {
    double n1 = Double.parseDouble(num1.getText().toString());
    double n2 = Double.parseDouble(num2.getText().toString());
    result.setText("Result: " + (n1 * n2));
});
btnDiv.setOnClickListener(v -> {
    double n1 = Double.parseDouble(num1.getText().toString());
    double n2 = Double.parseDouble(num2.getText().toString());
    result.setText("Result: " + (n1 / n2));
});
Group B
How can you use string and string array in android? Explain.

Strings and String Arrays are stored in res/values/strings.xml to support internationalization and easy maintenance.

Defining in XML:

<resources>
    <string name="app_name">My App</string>
    <string-array name="colors">
        <item>Red</item>
        <item>Green</item>
        <item>Blue</item>
    </string-array>
</resources>

Accessing in Java:

String appName = getResources().getString(R.string.app_name);
String[] colors = getResources().getStringArray(R.array.colors);
Group B
How to use spinner in android? Explain with a suitable example.

A Spinner provides a quick way to select one value from a set. In the default state, a spinner shows its currently selected value.

Example Implementation:

<!-- activity_main.xml -->
<Spinner android:id="@+id/mySpinner" android:layout_width="match_parent" android:layout_height="wrap_content" />

// MainActivity.java
Spinner spinner = findViewById(R.id.mySpinner);
String[] items = {"Android", "iOS", "Windows", "Harmony"};
ArrayAdapter<String> adapter = new ArrayAdapter<>(this, android.R.layout.simple_spinner_item, items);
adapter.setDropDownViewResource(android.R.layout.simple_spinner_dropdown_item);
spinner.setAdapter(adapter);

spinner.setOnItemSelectedListener(new OnItemSelectedListener() {
    @Override
    public void onItemSelected(AdapterView<?> parent, View view, int pos, long id) {
        Toast.makeText(getApplicationContext(), "Selected: " + items[pos], Toast.LENGTH_SHORT).show();
    }
    @Override
    public void onNothingSelected(AdapterView<?> parent) {}
});

What is the difference between android:layout_gravity and android:gravity attribute in linear layout? Create application to convert KM to Meters using Table Layout.
  • android:gravity: Specifies how to align the content within the view (e.g., text inside a TextView).
  • android:layout_gravity: Specifies how to align the entire view within its parent layout.

KM to Meters (Table Layout):

<!-- activity_main.xml -->
<TableLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent" android:layout_height="match_parent" android:stretchColumns="1">
    <TableRow>
        <TextView android:text="Kilometers:" />
        <EditText android:id="@+id/etKM" android:inputType="numberDecimal" />
    </TableRow>
    <TableRow>
        <Button android:id="@+id/btnConvert" android:text="Convert" />
        <TextView android:id="@+id/tvMeters" android:text="0 Meters" />
    </TableRow>
</TableLayout>

// Java logic
btnConvert.setOnClickListener(v -> {
    double km = Double.parseDouble(etKM.getText().toString());
    tvMeters.setText((km * 1000) + " Meters");
});

List and explain any five widgets used in android programming.
  1. TextView: Used to display text to the user.
  2. EditText: A text field that allows the user to enter and edit text.
  3. Button: A push-button that triggers an action when clicked.
  4. ImageView: Used to display image resources (e.g., bitmaps, drawables).
  5. CheckBox: A type of button that can be either checked or unchecked, often used in lists of options where multiple selections are allowed.
Group B
Differentiate RelativeLayout and ConstraintLayout with suitable example.
  • RelativeLayout: Views are positioned relative to the parent or to other sibling views (e.g., "to right of Button A"). It is simpler for flat hierarchies but can get complex.
  • ConstraintLayout: More advanced and flexible version of RelativeLayout. It allows you to build complex layouts with a flat view hierarchy. It uses "constraints" to define the position of a view relative to other views and the parent. It is optimized for performance and works well with the visual editor.

Develop an android application to calculate perimeter of rectangle using LinearLayout.
<!-- activity_main.xml -->
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent" android:layout_height="match_parent"
    android:orientation="vertical" android:padding="16dp">
    <EditText android:id="@+id/etLength" android:hint="Length" android:inputType="numberDecimal" />
    <EditText android:id="@+id/etWidth" android:hint="Width" android:inputType="numberDecimal" />
    <Button android:id="@+id/btnCalc" android:text="Calculate Perimeter" />
    <TextView android:id="@+id/tvRes" android:text="Result: " android:textSize="18sp" />
</LinearLayout>

// Java logic: perimeter = 2 * (length + width)
btnCalc.setOnClickListener(v -> {
    double l = Double.parseDouble(etLength.getText().toString());
    double w = Double.parseDouble(etWidth.getText().toString());
    tvRes.setText("Perimeter: " + (2 * (l + w)));
});
