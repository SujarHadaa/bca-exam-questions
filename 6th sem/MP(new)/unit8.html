What are different types of UI controls used in IOS programming? Write a swift program to find the sum of all elements of a 1-D array.

iOS UI Controls:

  • UILabel: Displays static text.
  • UIButton: Triggers actions when tapped.
  • UITextField: Allows single-line text input.
  • UISwitch: A toggle switch for binary choices.
  • UISlider: Allows selecting a value from a continuous range.
  • UIImageView: Displays images.
  • UITableView: Displays a list of items in a single column.

Swift Code (Sum of 1-D Array):

let numbers = [10, 20, 30, 40, 50]
var sum = 0
for num in numbers {
    sum += num
}
print("Sum is: \(sum)")
Group B
What do you mean by View hierarchy in IOS programming? Write a swift function to calculate the factorial.

View Hierarchy: In iOS, views are arranged in a tree-like structure. Every view has exactly one superview (except the root view) and can have multiple subviews. When a superview is moved or hidden, its subviews are also affected. This hierarchy determines the layout and the order in which views are rendered and receive events.

Swift Code (Factorial Function):

func factorial(n: Int) -> Int {
    if n == 0 { return 1 }
    var res = 1
    for i in 1...n {
        res *= i
    }
    return res
}
print(factorial(n: 5)) // Output: 120
Group B
What do you mean by storyboard? Write a Swift program to find the largest and smallest number among three numbers using a ternary operator.

Storyboard: A Storyboard is a visual representation of the user interface of an iOS application, showing screens of content and the transitions between them. It allows developers to design the UI and define the flow of the app without writing much code.

Swift Code (Largest and Smallest of 3 Numbers):

let a = 10, b = 25, c = 15
let largest = (a > b) ? ((a > c) ? a : c) : ((b > c) ? b : c)
let smallest = (a < b) ? ((a < c) ? a : c) : ((b < c) ? b : c)

print("Largest: \(largest), Smallest: \(smallest)")
Group B
Define ViewController.

ViewController (UIViewController): It is a fundamental building block of any iOS app. It manages a single view hierarchy (usually one screen) and acts as the "glue" between the Model (data) and the View (UI). Its responsibilities include updating the UI, handling user interactions, and managing memory.
