Unit 6: Socket for Clients

Socket Definition and TCP vs. UDP

Define Socket.

A socket is one endpoint of a two-way communication link between two programs running on the network. A socket is a combination of an IP address and a port number. The java.net.Socket class represents a client-side socket. When you create a socket, you specify the hostname and port of the server you want to connect to. Once the connection is established, you can get input and output streams from the socket to communicate with the server.

Differentiate between TCP and UDP Sockets.

Feature TCP (Transmission Control Protocol) UDP (User Datagram Protocol)
Connection Connection-oriented Connectionless
Reliability Reliable (guarantees delivery, order, and no duplication) Unreliable (no guarantee of delivery or order)
Speed Slower (due to overhead of reliability checks) Faster (less overhead)
Header Size 20 bytes 8 bytes
Use Cases Web browsing (HTTP), email (SMTP), file transfer (FTP) Video streaming, online gaming, DNS, VoIP
Java Classes Socket, ServerSocket DatagramSocket, DatagramPacket

Key Points to Remember: TCP vs. UDP

  • TCP: Reliable, ordered, connection-based. Think of it like a phone call. Use for things that must be perfect, like file transfers.
  • UDP: Fast, lightweight, connectionless. Think of it like sending a postcard. Use for things where speed is more important than perfect reliability, like streaming or gaming.

Define a secure socket.

A secure socket is a type of socket that provides a secure communication channel using cryptographic protocols like SSL (Secure Sockets Layer) or TLS (Transport Layer Security). It encrypts the data being transmitted between the client and the server, ensuring confidentiality and integrity. In Java, secure sockets are created using the javax.net.ssl.SSLSocket class, which is a subclass of java.net.Socket.

Java Program to Create a Secure Client Socket

This program shows how to create a secure client socket that connects to a secure server (like a website with HTTPS) and sends a simple HTTP request.


import javax.net.ssl.SSLSocket;
import javax.net.ssl.SSLSocketFactory;
import java.io.*;

public class SecureClientSocket {

    public static void main(String[] args) {
        String host = "www.google.com";
        int port = 443; // Standard HTTPS port

        try {
            // Get a factory for creating secure sockets
            SSLSocketFactory factory = (SSLSocketFactory) SSLSocketFactory.getDefault();

            // Create a secure socket and connect it
            try (SSLSocket socket = (SSLSocket) factory.createSocket(host, port)) {

                System.out.println("Connected to " + host + " on port " + port);

                // Enable all available cipher suites
                socket.setEnabledCipherSuites(socket.getSupportedCipherSuites());

                // Start the SSL handshake
                socket.startHandshake();

                // Send an HTTP GET request
                PrintWriter out = new PrintWriter(new BufferedWriter(new OutputStreamWriter(socket.getOutputStream())));
                out.println("GET / HTTP/1.1");
                out.println("Host: " + host);
                out.println();
                out.flush();

                // Read the response
                BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
                String line;
                while ((line = in.readLine()) != null) {
                    System.out.println(line);
                    // Break after a few lines for brevity
                    if (line.contains("") || line.contains("")) {
                        break;
                    }
                }
            }

        } catch (IOException e) {
            System.err.println("Error creating or communicating with the secure socket.");
            e.printStackTrace();
        }
    }
}

Key Points to Remember: Secure Sockets

  • Encrypts communication between client and server.
  • Uses SSL/TLS protocols.
  • In Java, use SSLSocketFactory to create an SSLSocket.
  • Connects to the server's secure port (usually 443 for HTTPS).

Displaying Socket Information

Java Program to Display Socket Information

This program connects to a server and then displays information about the connection, including the remote address and port, and the local address and port.


import java.io.IOException;
import java.net.Socket;
import java.net.UnknownHostException;

public class SocketInfo {

    public static void main(String[] args) {
        String host = "www.google.com";
        int port = 80; // Standard HTTP port

        try (Socket socket = new Socket(host, port)) {
            System.out.println("Successfully connected to " + host + " on port " + port);
            System.out.println("----------------------------------");

            // Display socket information
            System.out.println("Remote Address: " + socket.getInetAddress().getHostAddress());
            System.out.println("Remote Port: " + socket.getPort());
            System.out.println("Local Address: " + socket.getLocalAddress().getHostAddress());
            System.out.println("Local Port: " + socket.getLocalPort());

        } catch (UnknownHostException e) {
            System.err.println("Error: Don't know about host " + host);
        } catch (IOException e) {
            System.err.println("Error: Couldn't get I/O for the connection to " + host);
        }
    }
}

Key Points to Remember: Socket Information

  • Create a Socket object to connect to a server.
  • getInetAddress(): Gets the remote server's IP address info.
  • getPort(): Gets the remote server's port.
  • getLocalAddress(): Gets your machine's IP address used for the connection.
  • getLocalPort(): Gets the port on your machine used for the connection.
