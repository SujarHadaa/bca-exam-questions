Unit 3: URLs and URIs

The URL Class

Define URL class.

The URL class in Java represents a Uniform Resource Locator, which is a pointer to a "resource" on the World Wide Web. A resource can be something as simple as a file or a directory, or it can be a reference to a more complicated object, such as a query to a database or a search engine.

The URL class provides a simple, high-level API to access resources on the internet. Once you create a URL object, you can use it to open a connection to the resource it points to and retrieve its content.

Key Points to Remember: URL Class

  • Represents a web address (e.g., "http://www.example.com").
  • Provides methods to parse and access different parts of the URL.
  • Acts as a starting point for downloading web content.

Program to Split a URL

Java Program to Show the Parts of a URL

This program creates a URL object and then uses its getter methods to extract and display its various components, such as the protocol, host, port, and path.


import java.net.MalformedURLException;
import java.net.URL;

public class URLSplitter {

    public static void main(String[] args) {
        try {
            // Create a URL object
            URL url = new URL("https://www.example.com:8080/path/to/file.html?query=123#fragment");

            System.out.println("URL: " + url.toString());
            System.out.println("---------------------------------");

            // Split the URL into its components
            System.out.println("Protocol: " + url.getProtocol());
            System.out.println("Host: " + url.getHost());
            System.out.println("Port: " + url.getPort());
            System.out.println("Default Port: " + url.getDefaultPort());
            System.out.println("Path: " + url.getPath());
            System.out.println("File: " + url.getFile());
            System.out.println("Query: " + url.getQuery());
            System.out.println("Reference (Fragment): " + url.getRef());

        } catch (MalformedURLException e) {
            System.err.println("Error: The URL is not valid.");
            e.printStackTrace();
        }
    }
}

Key Points to Remember: Splitting a URL

  • Create a URL object from a URL string.
  • Use methods like getProtocol(), getHost(), getPort(), getPath(), getQuery(), and getRef().
  • Always wrap URL creation in a try-catch block for MalformedURLException.

The URLConnection Class

Define URLConnection Class.

The URLConnection class is an abstract class that represents a communication link between an application and a URL. It is a more general and powerful mechanism than the URL class for accessing web resources. While a URL object just points to a resource, a URLConnection object can be used to interact with it, allowing you to send data (for POST requests) and inspect the response headers and content.

You don't create a URLConnection directly. Instead, you get an instance by calling the openConnection() method on a URL object. The actual type of the connection object returned (e.g., HttpURLConnection) depends on the protocol of the URL.

Key Points to Remember: URLConnection Class

  • Represents an active connection to a web resource.
  • Obtained by calling url.openConnection().
  • Allows you to read response headers and content.
  • Allows you to configure the connection (e.g., set timeouts, add request headers).

Program to Read Response Headers

Java Program to Read Headers Using Header-Specific Methods

This program connects to a URL and uses the URLConnection object to read and display various response headers, such as the content type, content length, and date.


import java.io.IOException;
import java.net.URL;
import java.net.URLConnection;
import java.util.Date;

public class HeaderReader {

    public static void main(String[] args) {
        try {
            URL url = new URL("https://www.google.com");
            URLConnection connection = url.openConnection();

            // Connect to the resource
            connection.connect();

            System.out.println("Response Headers:");
            System.out.println("-------------------");

            // Use header-specific methods
            System.out.println("Content-Type: " + connection.getContentType());
            System.out.println("Content-Length: " + connection.getContentLength());
            System.out.println("Content-Encoding: " + connection.getContentEncoding());
            System.out.println("Date: " + new Date(connection.getDate()));
            System.out.println("Last-Modified: " + new Date(connection.getLastModified()));
            System.out.println("Expiration: " + new Date(connection.getExpiration()));

        } catch (IOException e) {
            System.err.println("Error connecting to the URL or reading headers.");
            e.printStackTrace();
        }
    }
}

Key Points to Remember: Reading Headers

  • Get a URLConnection from a URL object.
  • Call connect() to establish the connection.
  • Use methods like getContentType(), getContentLength(), and getDate().
  • Dates are returned as long values (milliseconds since epoch), so convert them to Date objects for readability.
