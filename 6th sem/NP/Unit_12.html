Unit 12: Remote Method Invocation (RMI)

RMI (Remote Method Invocation)

This topic is covered by multiple questions.

What is RMI?

Remote Method Invocation (RMI) is a Java API that allows an object running in one Java Virtual Machine (JVM) to invoke methods on an object running in another JVM. RMI provides the mechanism by which the server and client can communicate and pass information back and forth. It is a higher-level abstraction of network programming that allows developers to create distributed applications without having to deal with the low-level details of sockets and data serialization.

What is an RMI Server?

An RMI server is a program that creates one or more remote objects, makes them available for clients to call, and waits for incoming client requests. The core of an RMI server is the RMI registry, which is a simple server-side namespace that allows clients to look up remote objects by a registered name.

The RMI Architecture

RMI is based on a layered architecture:

  1. Stub and Skeleton Layer: The stub (on the client side) and the skeleton (on the server side) are responsible for intercepting method calls made by the client and redirecting them to the remote RMI service. The stub acts as a proxy for the remote object, and the skeleton is the corresponding object on the server side.
  2. Remote Reference Layer: This layer manages the references between the client and the remote object.
  3. Transport Layer: This is the low-level layer that handles the actual transportation of data over the network, typically using TCP/IP.

Key Points to Remember: RMI

  • Java-to-Java only: Allows calling methods on objects in different JVMs.
  • High-level abstraction: Hides socket programming details.
  • RMI Registry: A phonebook for finding remote objects.
  • Stub/Skeleton: Proxies that handle the communication between client and server.
