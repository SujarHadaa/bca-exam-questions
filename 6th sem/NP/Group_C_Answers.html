a) Differentiating URL and URI

URI (Uniform Resource Identifier) is a string of characters that unambiguously identifies a particular resource. It's a general concept. A URI can be a name, a locator, or both.

URL (Uniform Resource Locator) is a specific type of URI that not only identifies a resource but also provides a means of locating it by describing its primary access mechanism (i.e., its network location).

The key difference: Every URL is a URI, but not every URI is a URL. A URL specifies *how* to get to a resource (e.g., http://, ftp://). A URI can just be a unique name for a resource without specifying its location, like a URN (Uniform Resource Name), for example, urn:isbn:0451450523, which identifies a book but doesn't tell you where to find it.

In Java, the java.net.URI class is a more general representation and can handle all types of URIs. The java.net.URL class is specifically for URLs and provides methods to open connections and retrieve content.

Key Points to Remember: URL vs. URI

  • URI: An identifier. Can be a name, a location, or both.
  • URL: A specific kind of URI that tells you the location of a resource.
  • Analogy: Your name is a URI (it identifies you). Your address is a URL (it tells people where to find you).

The Chat Server


// ChatServer.java
import java.io.*;
import java.net.*;

public class ChatServer {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(9999);
        System.out.println("Server started. Waiting for a client...");

        Socket clientSocket = serverSocket.accept();
        System.out.println("Client connected: " + clientSocket.getInetAddress().getHostAddress());

        BufferedReader in = new BufferedReader(new InputStreamReader(clientSocket.getInputStream()));
        PrintWriter out = new PrintWriter(clientSocket.getOutputStream(), true);
        BufferedReader consoleReader = new BufferedReader(new InputStreamReader(System.in));

        String inputLine;
        while (true) {
            // Read from client
            if (in.ready()) {
                inputLine = in.readLine();
                System.out.println("Client: " + inputLine);
                if (inputLine.equalsIgnoreCase("bye")) break;
            }

            // Write to client
            if (consoleReader.ready()) {
                String serverMsg = consoleReader.readLine();
                out.println(serverMsg);
                if (serverMsg.equalsIgnoreCase("bye")) break;
            }
        }

        System.out.println("Closing connection.");
        clientSocket.close();
        serverSocket.close();
    }
}

import java.net.*;

public class TwoWayServer {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(5000);
        System.out.println("Server is waiting for a client...");
        Socket clientSocket = serverSocket.accept();
        System.out.println("Client connected.");

        // Streams for communication
        DataInputStream dis = new DataInputStream(clientSocket.getInputStream());
        DataOutputStream dos = new DataOutputStream(clientSocket.getOutputStream());

        String clientMessage = "", serverMessage = "";

        while (!clientMessage.equals("stop")) {
            clientMessage = dis.readUTF();
            System.out.println("Client says: " + clientMessage);
            System.out.print("Enter your message: ");
            serverMessage = br.readLine();
            dos.writeUTF(serverMessage);
            dos.flush();
        }

        dis.close();
        dos.close();
        clientSocket.close();
        serverSocket.close();
    }
}

Write multithreaded TCP client and server socket programs for daytime service.

Multithreaded Daytime Server

A multithreaded server can handle multiple client requests simultaneously. When the server accepts a new client connection, it spawns a new thread to handle that client, and then immediately goes back to waiting for the next connection. This prevents new clients from being blocked while the server is busy with an existing one.

The Multithreaded Daytime Server


// MultithreadedDaytimeServer.java
import java.io.IOException;
import java.io.PrintWriter;
import java.net.ServerSocket;
import java.net.Socket;
import java.util.Date;

public class MultithreadedDaytimeServer {
    public static final int PORT = 1313;

    public static void main(String[] args) {
        try (ServerSocket serverSocket = new ServerSocket(PORT)) {
            System.out.println("Multithreaded Daytime Server started on port " + PORT);
            while (true) {
                Socket clientSocket = serverSocket.accept();
                // Create a new thread for each client
                new ClientHandler(clientSocket).start();
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

class ClientHandler extends Thread {
    private Socket clientSocket;

    public ClientHandler(Socket socket) {
        this.clientSocket = socket;
    }

    public void run() {
        try {
            System.out.println("Client connected: " + clientSocket.getInetAddress());
            PrintWriter out = new PrintWriter(clientSocket.getOutputStream(), true);
            out.println(new Date().toString());
            System.out.println("Sent time to client " + clientSocket.getInetAddress());
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            try {
                clientSocket.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}

What is the procedure for constructing and connecting Server sockets? Explain the Server Socket options with examples.

Procedure for Constructing and Connecting Server Sockets

  1. Construction: You create a ServerSocket object, specifying the port number you want it to listen on. For example: ServerSocket server = new ServerSocket(8080);. You can also specify a backlog (the maximum queue length for incoming connection requests) and an address to bind to if the machine has multiple IP addresses.
  2. Connecting (Accepting): You call the accept() method on the ServerSocket. This is a blocking call that waits until a client tries to connect. When a client connects, accept() returns a Socket object that represents the connection to that specific client. All further communication with that client happens through this new Socket object.

ServerSocket Options

ServerSocket options are set using methods on the ServerSocket object before it starts accepting connections.

  • SO_TIMEOUT: Sets a timeout on the accept() method. If no client connects within the specified time (in milliseconds), it throws a SocketTimeoutException. This is useful if you want the server to be able to do other things periodically instead of just waiting forever.
  • SO_REUSEADDR: This is a crucial option. It allows the server to bind to a port that is in a `TIME_WAIT` state from a previous, recently closed socket. Without this, you might have to wait a few minutes to restart a server on the same port, which is very inconvenient during development.
  • SO_RCVBUF: Sets the size of the underlying buffer for incoming data on the sockets that are created by this ServerSocket.

Example of Setting ServerSocket Options


import java.io.IOException;
import java.net.ServerSocket;
import java.net.Socket;
import java.net.SocketTimeoutException;

public class ServerSocketOptionsExample {
    public static void main(String[] args) {
        try {
            ServerSocket serverSocket = new ServerSocket(8080);

            // Set SO_TIMEOUT to 10 seconds (10000 ms)
            serverSocket.setSoTimeout(10000);
            System.out.println("Accept timeout set to 10 seconds.");

            // Enable SO_REUSEADDR
            serverSocket.setReuseAddress(true);
            System.out.println("SO_REUSEADDR enabled.");

            System.out.println("Waiting for a client...");
            Socket clientSocket = serverSocket.accept(); // This will time out after 10 seconds
            System.out.println("Client connected: " + clientSocket.getInetAddress());

            clientSocket.close();
            serverSocket.close();

        } catch (SocketTimeoutException e) {
            System.err.println("No client connected within 10 seconds. Server shutting down.");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Key Points to Remember: ServerSocket Procedure & Options

  • Construct: new ServerSocket(port).
  • Connect: Call accept() to wait for a client.
  • SO_TIMEOUT: Prevents accept() from blocking forever.
  • SO_REUSEADDR: Lets you restart the server quickly on the same port.

How blocking I/O is different from Non-Blocking I/O? Explain the basic mechanism of handling Buffers and Channels in Java with an example.

a) Blocking vs. Non-Blocking I/O

This is the same as a previous question. See the detailed comparison table in Question 2 of the Non-Blocking I/O unit.

In short: Blocking I/O waits, non-blocking I/O doesn't. A thread making a blocking call (e.g., read()) will sleep until the operation is complete. A thread making a non-blocking call will return immediately, either with data or with an indication that there is no data yet.

b) Mechanism of Handling Buffers and Channels

The basic mechanism in Java NIO is as follows:

  1. You have data that you want to read or write (e.g., in a file or coming from a network connection).
  2. You create a Channel to that source or destination (e.g., a FileChannel or a SocketChannel). The channel is the conduit for the data.
  3. You create a Buffer (e.g., a ByteBuffer). This is a fixed-size block of memory that will temporarily hold the data.
  4. Reading data: You call the channel's read(buffer) method. The channel reads data from the source and puts it into your buffer.
  5. Writing data: You first put your data into the buffer (using the buffer's put() method). Then you call the channel's write(buffer) method, which takes the data from your buffer and sends it to the destination.

The key is that you always work with the buffer as an intermediary. You never write directly to the channel or read directly from it.

Example: Reading from a File with a Channel and Buffer


import java.io.FileInputStream;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;

public class BufferChannelFileRead {
    public static void main(String[] args) throws Exception {
        FileInputStream fis = new FileInputStream("questions.txt");
        FileChannel channel = fis.getChannel();

        ByteBuffer buffer = ByteBuffer.allocate(1024);

        // Read from channel into buffer
        int bytesRead = channel.read(buffer);

        System.out.println("Read " + bytesRead + " bytes.");

        // Prepare buffer for reading
        buffer.flip();

        System.out.print("Buffer contents: ");
        while(buffer.hasRemaining()){
            System.out.print((char) buffer.get());
        }

        fis.close();
    }
}

Key Points to Remember: Buffers and Channels

  • Channel: The data pipe (to a file, socket, etc.).
  • Buffer: The data container.
  • Read process: Channel read() -> Buffer.
  • Write process: Your data put() -> Buffer, then Channel write() -> Destination.
  • flip(): The essential method to switch a buffer from write mode to read mode.

What are the features of DatagramPacket and DatagramSocket Class? Write a program to create a simple UDP client.

Features of DatagramPacket

  • Self-contained: Each packet holds its own data, destination address, and port.
  • Stateless: No connection is established; each packet is independent.
  • Constructors: Has two main constructors: one for sending (specifying data, address, port) and one for receiving (providing a buffer to be filled).
  • Data Handling: Provides methods like getData(), getLength(), getAddress(), and getPort() to access the information within the packet.

Features of DatagramSocket

  • Connectionless: Used for sending and receiving UDP packets.
  • Binding: Can be bound to a specific port to listen for incoming packets. If not specified, it binds to an available port.
  • Sending/Receiving: Uses send(DatagramPacket p) and receive(DatagramPacket p) methods. receive() is a blocking call.
  • Socket Options: Supports options like SO_TIMEOUT to prevent blocking indefinitely on a receive call.

BS to AD Date Converter (UDP)

This example uses a simplified conversion for demonstration purposes.

The BS to AD UDP Server


// BsToAdUdpServer.java
import java.io.IOException;
import java.net.DatagramPacket;
import java.net.DatagramSocket;
import java.net.InetAddress;

public class BsToAdUdpServer {
    public static void main(String[] args) throws IOException {
        try (DatagramSocket socket = new DatagramSocket(9877)) {
            System.out.println("BS to AD UDP Server is running...");
            byte[] receiveData = new byte[1024];

            while (true) {
                DatagramPacket receivePacket = new DatagramPacket(receiveData, receiveData.length);
                socket.receive(receivePacket);

                String bsDateStr = new String(receivePacket.getData(), 0, receivePacket.getLength()).trim();
                System.out.println("RECEIVED BS Date: " + bsDateStr);

                // Simple conversion logic (assuming format YYYY-MM-DD)
                String[] parts = bsDateStr.split("-");
                int bsYear = Integer.parseInt(parts[0]);
                int bsMonth = Integer.parseInt(parts[1]);
                int bsDay = Integer.parseInt(parts[2]);

                int adYear = bsYear - 56;
                int adMonth = bsMonth - 8;
                int adDay = bsDay - 17;

                // Basic adjustments for negative month/day
                if (adDay <= 0) {
                    adDay += 30; // Approximate
                    adMonth -= 1;
                }
                if (adMonth <= 0) {
                    adMonth += 12;
                    adYear -= 1;
                }

                String adDateStr = String.format("%04d-%02d-%02d", adYear, adMonth, adDay);

                InetAddress ipAddress = receivePacket.getAddress();
                int port = receivePacket.getPort();

                byte[] sendData = adDateStr.getBytes();
                DatagramPacket sendPacket = new DatagramPacket(sendData, sendData.length, ipAddress, port);
                socket.send(sendPacket);
            }
        }
    }

Explain the factory method with an example. Write a program to download a webpage.

a) The Factory Method Pattern

The Factory Method is a creational design pattern that provides an interface for creating objects in a superclass, but allows subclasses to alter the type of objects that will be created. It's called a "factory" because it's responsible for "manufacturing" objects.

Example: Imagine a logistics company. The main company might have a createTransport() method. A subclass like RoadLogistics would implement this method to create a Truck object, while a SeaLogistics subclass would implement it to create a Ship object. The client code doesn't need to know about trucks or ships; it just asks the factory to create a transport object.

In network programming, SSLSocketFactory is a great example. You don't create an SSLSocket directly with new. Instead, you get a factory object and call its createSocket() method. This pattern allows the framework to handle the complex details of creating and configuring the secure socket.

            URLConnection connection = url.openConnection();

            // Open streams to read from the URL and write to a file
            try (BufferedReader reader = new BufferedReader(new InputStreamReader(connection.getInputStream()));
                 PrintWriter writer = new PrintWriter(new FileWriter(outputFileName))) {

                System.out.println("Downloading from " + urlString + "...");
                String line;
                while ((line = reader.readLine()) != null) {
                    writer.println(line);
                }
                System.out.println("Successfully downloaded to " + outputFileName);
            }

        } catch (IOException e) {
            System.err.println("An error occurred during download.");
            e.printStackTrace();
        }
    }
}

Key Points to Remember: Webpage Downloader

  • Create a URL object from the webpage address.
  • Call openConnection() on the URL.
  • Get the InputStream from the connection to read the page content.
  • Use a FileWriter or PrintWriter to save the content to a local file.
  • Read from the input stream line-by-line and write to the file.

a) RMI Program to Calculate Area of a Triangle

1. The Remote Interface


// TriangleArea.java (Interface)
import java.rmi.Remote;
import java.rmi.RemoteException;

public interface TriangleArea extends Remote {
    double getArea(double base, double height) throws RemoteException;
}

2. The Implementation


// TriangleAreaImpl.java (Implementation)
import java.rmi.RemoteException;
import java.rmi.server.UnicastRemoteObject;

public class TriangleAreaImpl extends UnicastRemoteObject implements TriangleArea {
    public TriangleAreaImpl() throws RemoteException {
        super();
    }

    @Override
    public double getArea(double base, double height) throws RemoteException {
        return 0.5 * base * height;
    }
}

3. The RMI Server


// TriangleServer.java
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;

public class TriangleServer {
    public static void main(String[] args) {
        try {
            TriangleAreaImpl server = new TriangleAreaImpl();
            Registry registry = LocateRegistry.createRegistry(1099);
            registry.rebind("TriangleAreaServer", server);
            System.out.println("Triangle Area RMI Server is running...");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

4. The RMI Client


// TriangleClient.java
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;

public class TriangleClient {
    public static void main(String[] args) {
        try {
            Registry registry = LocateRegistry.getRegistry("localhost", 1099);
            TriangleArea server = (TriangleArea) registry.lookup("TriangleAreaServer");
            double area = server.getArea(10, 5);
            System.out.println("Area of the triangle is: " + area);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

b) Program to List All Network Interfaces


// ListNetworkInterfaces.java
import java.net.NetworkInterface;
import java.util.Collections;
import java.util.Enumeration;

public class ListNetworkInterfaces {
    public static void main(String[] args) throws Exception {
        Enumeration nets = NetworkInterface.getNetworkInterfaces();
        for (NetworkInterface netint : Collections.list(nets)) {
            System.out.printf("Display name: %s\n", netint.getDisplayName());
            System.out.printf("Name: %s\n", netint.getName());
            System.out.printf("Up? %s\n", netint.isUp());
            System.out.printf("Loopback? %s\n", netint.isLoopback());
            System.out.printf("Virtual? %s\n", netint.isVirtual());
            System.out.printf("Supports multicast? %s\n", netint.supportsMulticast());
            System.out.println("---------------------------------");
        }
    }
}

Key Points to Remember:

  • RMI Area Calculator: Follows the standard RMI pattern: Interface, Implementation, Server, Client. The remote method takes base and height and returns the calculated area.
  • List Interfaces: Use NetworkInterface.getNetworkInterfaces() to get an Enumeration of all interfaces. Loop through it and use methods like getDisplayName(), getName(), and isUp() to get details about each one.
