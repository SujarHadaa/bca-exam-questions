Unit 9: Non-blocking I/O

Blocking vs. Non-blocking I/O

This question is repeated 2 times.

Define Non-blocking I/O.

Non-blocking I/O is an approach to input/output operations where a call to read or write data returns immediately, regardless of whether the operation can be completed. If the operation cannot be completed at that moment (e.g., no data is available to be read), the call returns a status indicating that, allowing the program to continue with other tasks. This is in contrast to blocking I/O, where the call would wait until the operation is complete.

How is blocking I/O different from Non-Blocking I/O?

Feature Blocking I/O Non-blocking I/O
Thread Behavior The calling thread is blocked until the I/O operation completes. The calling thread is not blocked; the I/O call returns immediately.
Efficiency Can be inefficient for handling multiple connections, as it requires a separate thread for each connection to avoid blocking the entire application. Highly efficient for handling many connections with a single thread, as the thread can switch to other tasks while waiting for I/O.
Complexity Simpler to program and understand. More complex, as it requires mechanisms (like selectors) to know when an I/O operation is ready to be performed.
Use Cases Simple client applications, or servers with a low number of concurrent connections. High-performance servers that need to handle thousands of concurrent connections, like chat servers or web servers.

Key Points to Remember: Blocking vs. Non-blocking I/O

  • Blocking: Waits for I/O to finish. Simple but can be slow with many clients.
  • Non-blocking: Doesn't wait. More complex but very efficient for high-concurrency servers.

Central Abstractions of Java NIO

Explain the central abstractions of Java NIO.

Java New I/O (NIO) is an alternative I/O API for Java that provides support for non-blocking I/O. Its central abstractions are:

  • Channels: A channel represents a connection to an entity that is capable of performing I/O operations, such as a file or a socket. It is the gateway through which data is transferred. You read data from a channel into a buffer, and you write data from a buffer into a channel.
  • Buffers: A buffer is a fixed-size block of memory that is used to hold data that is being transferred to or from a channel. All I/O in NIO is done through buffers. You don't write directly to a channel; you write to a buffer, which is then written to the channel.
  • Selectors: A selector is an object that can be used to monitor multiple channels for I/O events (like connection ready, data available for reading, or channel ready for writing). This is the key to non-blocking I/O, as it allows a single thread to manage multiple channels efficiently.

Key Points to Remember: Java NIO Abstractions

  • Channels: The pipes that data flows through (like a socket or file).
  • Buffers: The containers for data that is being sent or received through channels.
  • Selectors: The traffic cop that tells you which channels are ready for I/O, enabling non-blocking operations.

Buffers and Buffer Handling

Define Buffer.

A buffer is a temporary storage area for data. In Java NIO, a Buffer is an object that holds a fixed amount of data that can be sent to or received from a channel. It has a certain capacity and contains four key properties that control data handling:

  • Capacity: The maximum number of elements the buffer can hold. This is set at creation and cannot be changed.
  • Limit: The index of the first element that should not be read or written. It effectively marks the end of the active data in the buffer.
  • Position: The index of the next element to be read or written.
  • Mark: A remembered position. Calling mark() sets the mark to the current position, and reset() sets the position back to the mark.

Explain buffer handling methods.

The main methods for handling a buffer revolve around changing its state for reading and writing:

  • put(): Writes data into the buffer at the current position and increments the position.
  • get(): Reads data from the buffer at the current position and increments the position.
  • flip(): Prepares the buffer for reading. It sets the limit to the current position and then resets the position to zero. This is done after you have finished writing to the buffer and want to read from it.
  • rewind(): Resets the position to zero, allowing you to reread the data in the buffer. The limit remains unchanged.
  • clear(): Prepares the buffer for writing. It resets the position to zero and sets the limit to the capacity. It does not erase the data in the buffer, but you will be overwriting it.
  • compact(): Shifts the unread data to the beginning of the buffer, making space for new data to be written at the end.

Java Program Demonstrating Buffer and Channel Handling

This example shows how to use a FileChannel and a ByteBuffer to read data from a file.


import java.io.FileInputStream;
import java.io.IOException;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;

public class BufferChannelExample {

    public static void main(String[] args) {
        try (FileInputStream fis = new FileInputStream("questions.txt");
             FileChannel channel = fis.getChannel()) {

            // Create a buffer with a capacity of 1024 bytes
            ByteBuffer buffer = ByteBuffer.allocate(1024);

            System.out.println("Reading data from the file...");

            // Read data from the channel into the buffer
            int bytesRead = channel.read(buffer);

            while (bytesRead != -1) {
                // Flip the buffer to prepare it for reading
                buffer.flip();

                // Read the data from the buffer
                while (buffer.hasRemaining()) {
                    System.out.print((char) buffer.get());
                }

                // Clear the buffer to prepare it for writing
                buffer.clear();

                // Read more data from the channel
                bytesRead = channel.read(buffer);
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Key Points to Remember: Buffer Handling

  • put(): Write into the buffer.
  • flip(): Switch from writing to reading mode.
  • get(): Read from the buffer.
  • clear(): Switch back to writing mode.
  • The cycle is typically: clear() -> put() -> flip() -> get() -> clear().
